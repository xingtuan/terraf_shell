# 문제 해결

이 문서는 한국어 요약입니다. 전체 체크리스트는 [../TROUBLESHOOTING.md](../TROUBLESHOOTING.md)를 기준으로 합니다.

## 설치 실패

다음 오류가 나오면:

```text
Existing repo has local modifications
```

배포 디렉터리의 변경을 확인합니다.

```bash
git -C /var/www/terraf_shell status --short
```

변경을 버려도 되는 경우에만 실행합니다.

```bash
sudo env RESET_WORKTREE=1 RUN_SEED=0 bash auto_deploy.sh your-domain-or-ip
```

## 프론트엔드 접속 실패

```bash
sudo systemctl status terraf-frontend
sudo journalctl -u terraf-frontend -f
curl -I http://127.0.0.1:3000/
```

`B2C_frontend/.env.local` 확인:

```dotenv
NEXT_PUBLIC_API_BASE_URL=/api
NEXT_SERVER_API_BASE_URL=http://127.0.0.1:8000/api
API_PROXY_TARGET=http://127.0.0.1:8000
```

## API 또는 관리자 접속 실패

```bash
curl -I http://127.0.0.1:8000/up
sudo systemctl status php8.3-fpm
sudo nginx -t
sudo tail -f /var/log/nginx/error.log
```

관리자 주소:

```text
http://your-domain-or-ip:8000/admin
```

## 이미지가 보이지 않음

Local storage:

```bash
cd /var/www/terraf_shell/B2C_backend
php artisan storage:link
ls -l public/storage
```

Azure storage:

- Storage Settings에서 connection test 실행.
- Upload test 실행.
- Container, key, public URL, SAS TTL 확인.

## Cart / Checkout 문제

확인 항목:

- 상품과 variant 상태.
- SKU 재고와 inventory policy.
- Guest Checkout feature flag.
- Shipping Settings 결과.
- Tax Settings.
- `B2C_backend/storage/logs/laravel.log`.

## GST / 배송 설정 미반영

Tax Settings 또는 Shipping Settings 저장 후 캐시를 정리합니다.

```bash
cd /var/www/terraf_shell/B2C_backend
php artisan optimize:clear
```

NZ Post 견적은 NZ Post Settings의 인증 정보가 필요합니다. `auto` 모드에서는 NZ Post 실패 시 수동 요금으로 fallback 합니다.

## 500 / 502 / 404

- 500: Laravel 로그, `.env`, APP_KEY, 데이터베이스, storage 권한 확인.
- 502: `terraf-frontend`, Laravel `/up`, PHP-FPM, Nginx upstream 확인.
- 404: Nginx site, Next route, Laravel route cache 확인.

## Queue와 Scheduler

```bash
sudo supervisorctl status terraf-queue:*
tail -f /var/www/terraf_shell/B2C_backend/storage/logs/queue-worker.log
cat /etc/cron.d/terraf-scheduler
sudo systemctl status cron
```

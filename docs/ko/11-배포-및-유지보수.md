## 권장 배포 방식

현재 프로젝트는 Ubuntu / Debian 단일 서버에서 루트의 `auto_deploy.sh` 사용을 권장합니다.

```bash
sudo bash auto_deploy.sh your-domain-or-ip
```

운영 업데이트 예시:

```bash
sudo env RUN_SEED=0 bash auto_deploy.sh your-domain-or-ip
```

스크립트는 PHP 8.3, Composer, Node.js 20, pnpm, MySQL, Nginx, PHP-FPM, Supervisor, Cron을 설치하고 코드 가져오기, 의존성 설치, 마이그레이션, 선택적 시드, 프론트엔드 빌드, 큐와 스케줄러 설정을 수행합니다.

## 접속 주소

배포 후 기본 주소:

- 프론트엔드: `http://your-domain-or-ip/`
- API: `http://your-domain-or-ip/api/`
- 관리자: `http://your-domain-or-ip:8000/admin`
- 헬스 체크: `http://your-domain-or-ip:8000/up`

데이터베이스 정보와 주요 명령은 다음 파일에 저장됩니다.

```text
/root/terraf-install/credentials.txt
```

## 재배포

일반 재배포:

```bash
sudo env RUN_SEED=0 bash auto_deploy.sh your-domain-or-ip
```

로컬 변경이 있다는 오류가 나오면 먼저 확인합니다.

```bash
git -C /var/www/terraf_shell status --short
```

로컬 변경을 버려도 되는 경우에만 실행합니다.

```bash
sudo env RUN_SEED=0 RESET_WORKTREE=1 bash auto_deploy.sh your-domain-or-ip
```

`RESET_WORKTREE=1` 은 Git hard reset을 수행하고 untracked file을 삭제합니다.

## 유지보수 포인트

- 인수인계 전 기본 관리자 비밀번호를 변경합니다.
- 운영 업데이트는 보통 `RUN_SEED=0` 으로 실행합니다.
- 배포 전 MySQL과 업로드 파일을 백업합니다.
- `terraf-frontend`, `terraf-queue`, Nginx, PHP-FPM, MySQL 상태를 확인합니다.
- HTTPS 적용 후 `APP_URL`, `FRONTEND_URL`, `NEXT_PUBLIC_SITE_URL`, CORS, Sanctum 도메인을 갱신합니다.
- Storage Settings에서 local / Azure storage를 테스트합니다.
- Tax Settings와 Shipping Settings에서 GST와 배송비를 확인합니다.

## 주요 명령

```bash
sudo systemctl status terraf-frontend
sudo supervisorctl status terraf-queue:*
sudo systemctl status nginx
sudo systemctl status php8.3-fpm
sudo systemctl status mysql
```

백엔드 캐시 정리:

```bash
cd /var/www/terraf_shell/B2C_backend
php artisan optimize:clear
php artisan config:cache
```

프론트엔드 재빌드:

```bash
cd /var/www/terraf_shell/B2C_frontend
pnpm build
sudo systemctl restart terraf-frontend
```

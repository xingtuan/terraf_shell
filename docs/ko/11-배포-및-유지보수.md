# 11 — 배포 및 유지보수

## 개요

이 가이드는 OXP 플랫폼의 로컬 개발 환경 설정, 프로덕션 배포, 지속적인 유지보수 절차를 다룹니다.

## 2. 로컬 개발 환경 설정

### 2.1 백엔드 설정

```bash
# 1. 저장소 복제
git clone {repository-url}
cd terraf/B2C_backend

# 2. PHP 의존성 설치
composer install

# 3. 환경 파일 복사
cp .env.example .env

# 4. 애플리케이션 키 생성
php artisan key:generate

# 5. .env 설정
# 로컬 데이터베이스 자격증명 및 설정으로 .env 편집

# 6. 데이터베이스 마이그레이션 실행
php artisan migrate

# 7. 초기 콘텐츠 시딩 (선택 사항)
php artisan db:seed

# 8. 스토리지 심볼릭 링크 생성 (로컬 디스크 스토리지용)
php artisan storage:link

# 9. 개발 서버 시작
php artisan serve
```

백엔드는 `http://127.0.0.1:8000`에서 접근할 수 있습니다.

### 2.2 프론트엔드 설정

```bash
# 프론트엔드 디렉토리로 이동
cd terraf/B2C_frontend

# 의존성 설치
npm install

# 환경 설정
# .env.local 생성 후 다음 내용 추가:
# NEXT_PUBLIC_API_URL=http://127.0.0.1:8000/api

# 개발 서버 시작
npm run dev
```

프론트엔드는 `http://localhost:3000`에서 접근할 수 있습니다.

### 2.3 개발 환경 참고 사항

- 개발 시 `MAIL_MAILER=log` 설정 — 이메일은 `storage/logs/laravel.log`에 기록됩니다.
- Azure 대신 로컬 스토리지를 사용하려면 `FILESYSTEM_DISK=public` 설정
- 워커 없이 작업을 동기적으로 실행하려면 `QUEUE_CONNECTION=sync` 설정
- 기본 관리자 자격증명은 `UserSeeder`에서 설정됩니다 — 값은 시더를 확인하세요.

## 3. 프로덕션 배포

### 3.1 백엔드 프로덕션 배포

```bash
# 1. 환경 설정
APP_ENV=production
APP_DEBUG=false

# 2. 의존성 설치 (개발 패키지 제외)
composer install --no-dev --optimize-autoloader

# 3. 애플리케이션 키 설정 (새 환경인 경우)
php artisan key:generate

# 4. 프로덕션용 .env 설정
# - DB 자격증명 설정
# - APP_URL을 프로덕션 URL로 설정
# - FRONTEND_URL을 프로덕션 프론트엔드 URL로 설정
# - CORS_ALLOWED_ORIGINS를 프로덕션 프론트엔드 URL로 설정
# - AZURE 스토리지 자격증명 설정
# - MAIL 설정
# - QUEUE_CONNECTION=database 설정 (비동기 작업용)

# 5. 마이그레이션 실행
php artisan migrate --force

# 6. (선택) 초기 데이터 시딩
php artisan db:seed

# 7. Laravel 최적화
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan optimize

# 8. 파일 권한 설정
chmod -R 775 storage
chmod -R 775 bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

# 9. 스토리지 링크 생성 (로컬 디스크 사용 시)
php artisan storage:link
```

### 3.2 프론트엔드 프로덕션 빌드

```bash
cd B2C_frontend

# 의존성 설치
npm install --production

# 프로덕션 환경 설정
# .env.production 생성 후 다음 내용 추가:
# NEXT_PUBLIC_API_URL=https://api.yourproductiondomain.com

# 빌드
npm run build

# 프로덕션 서버 시작
npm run start
# 또는 .next/ 출력물을 호스팅 공급자에 배포
```

### 3.3 웹 서버 설정

**Laravel 백엔드를 위한 Nginx 예시:**

```nginx
server {
    listen 80;
    server_name api.yoursite.com;
    root /var/www/B2C_backend/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

## 4. 큐 워커 설정

플랫폼은 백그라운드 작업(주로 알림)을 위해 Laravel 큐 시스템을 사용합니다. 프로덕션에서는 큐 워커가 지속적으로 실행되어야 합니다.

### 4.1 큐 워커 실행

```bash
php artisan queue:work --sleep=3 --tries=3
```

### 4.2 Supervisor 사용 (프로덕션 권장)

Supervisor 설정 파일 생성:

```ini
[program:oxp-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/B2C_backend/artisan queue:work --sleep=3 --tries=3
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/www/B2C_backend/storage/logs/worker.log
stopwaitsecs=3600
```

```bash
supervisorctl reread
supervisorctl update
supervisorctl start oxp-worker:*
```

## 5. 스토리지 설정

### 5.1 Azure Blob Storage (프로덕션)

1. Azure 스토리지 계정과 컨테이너를 만듭니다.
2. 필수 환경 변수를 설정합니다:
   - `AZURE_STORAGE_NAME`
   - `AZURE_STORAGE_KEY`
   - `AZURE_STORAGE_CONTAINER`
   - `AZURE_STORAGE_URL`
3. `FILESYSTEM_DISK=azure` 및 `COMMUNITY_UPLOAD_DISK=azure`로 설정합니다.

### 5.2 로컬 스토리지 (개발 / 단순 프로덕션)

```bash
# 심볼릭 링크 생성
php artisan storage:link

# 링크 생성 확인
ls -la public/storage
# 표시 예: storage -> /var/www/storage/app/public
```

`.env`에서 `FILESYSTEM_DISK=public`으로 설정합니다.

## 6. 유지보수 명령어

### 6.1 캐시 지우기

```bash
# 모든 캐시 지우기
php artisan optimize:clear

# 개별적으로:
php artisan config:clear
php artisan route:clear
php artisan view:clear
php artisan cache:clear
```

> **참고**: `.env` 변경 후에는 `optimize:clear`를 실행하세요.

### 6.2 마이그레이션 실행

```bash
# 마이그레이션 상태 확인
php artisan migrate:status

# 대기 중인 마이그레이션 실행
php artisan migrate

# 마지막 배치 롤백
php artisan migrate:rollback
```

### 6.3 큐 관리

```bash
# 실패한 작업 보기
php artisan queue:failed

# 모든 실패한 작업 재시도
php artisan queue:retry all

# 특정 작업 재시도
php artisan queue:retry {job-id}

# 실패한 작업 초기화
php artisan queue:flush
```

### 6.4 로그 관리

로그는 `storage/logs/laravel.log`에 저장됩니다.

```bash
# 로그 파일 지우기
php artisan log:clear

# 또는 수동으로:
truncate -s 0 storage/logs/laravel.log
```

### 6.5 스토리지 링크 확인

```bash
php artisan storage:link
```

## 8. 데이터베이스 백업 전략

### 8.1 일별 데이터베이스 백업

```bash
# 수동 백업
mysqldump -u {username} -p {database_name} > backup_$(date +%Y%m%d_%H%M%S).sql

# cron으로 자동화 (crontab에 추가)
0 2 * * * mysqldump -u {username} -p{password} {database} | gzip > /backups/db_$(date +\%Y\%m\%d).sql.gz
```

### 8.2 백업 보존 기간

- **일별 백업**: 30일 보존
- **주별 백업**: 12주 보존
- **월별 백업**: 12개월 보존
- 서버와 다른 위치에 백업을 저장하세요.

### 8.3 백업에서 복원

```bash
mysql -u {username} -p {database_name} < backup_file.sql
```

## 9. 미디어 백업 전략

**Azure Blob Storage** 사용 시:
- Azure는 내장 이중화를 제공합니다.
- **Azure Backup**을 활성화하거나 AzCopy를 사용하여 컨테이너를 정기적으로 스냅샷합니다.

**로컬 스토리지** 사용 시:
- 백업 루틴에 `storage/app/public/`을 포함합니다.
- 데이터베이스 백업과 함께 매일 백업합니다.

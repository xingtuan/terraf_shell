# 13 — 자주 묻는 질문 (FAQ)

---

## A섹션: 일반 사용자 질문

### A1. 웹사이트를 탐색하기 위해 계정이 필요한가요?

아니요. 계정 없이도 홈페이지, 소재 라이브러리, 기사, 커뮤니티 게시물, 제품 카탈로그 등 모든 공개 콘텐츠를 탐색할 수 있습니다. 구매, 커뮤니티 게시물 작성, 콘텐츠 상호작용(좋아요, 댓글), 게시물 저장을 하려면 계정이 필요합니다.

### A2. 웹사이트 언어를 어떻게 변경하나요?

상단 내비게이션 바의 언어 전환기를 사용하세요. 플랫폼은 **영어 (English)**, **한국어**, **중국어 간체 (简体中文)**를 지원합니다.

### A3. 계정을 만들지 않고 결제할 수 있나요?

예. 비회원 결제가 지원됩니다. 이메일 주소와 배송 정보를 입력해야 합니다. 나중에 결제 시 사용한 이메일 주소와 주문 번호를 통해 스토어 → 주문 페이지에서 주문을 조회할 수 있습니다.

### A4. 주문을 어떻게 추적하나요?

- **로그인 상태인 경우**: 계정 → 주문으로 이동하여 모든 주문을 확인합니다.
- **비회원으로 결제한 경우**: 스토어 → 주문으로 이동하여 주문 번호(형식: `OXP-XXXXXX`)와 결제 시 사용한 이메일 주소를 입력합니다.

### A5. 스토어에서 사용하는 통화는 무엇인가요?

스토어는 **뉴질랜드 달러 (NZD)**로 운영됩니다. 가격에는 뉴질랜드 물품 서비스세(GST) 15%가 포함되어 있습니다.

### A6. 비밀번호를 어떻게 재설정하나요?

로그인 페이지에서 **비밀번호 찾기**를 클릭합니다. 이메일 주소를 입력하고 제출을 클릭합니다. 이메일로 비밀번호 재설정 링크가 전송됩니다. 링크를 클릭하고 새 비밀번호를 입력합니다. 링크는 짧은 시간 후 만료됩니다 — 필요한 경우 새로 요청하세요.

### A7. 왜 내 계정이 제한되었나요?

계정 제한은 커뮤니티 가이드라인 위반으로 인해 플랫폼 관리자가 적용합니다. 제한된 계정은 로그인하여 탐색할 수 있지만 게시물이나 댓글을 작성할 수 없습니다. 자세한 내용은 플랫폼 지원팀에 문의하세요.

### A8. 계정을 삭제할 수 있나요?

현재 사용자는 프론트엔드에서 직접 계정을 삭제할 수 없습니다. 계정 삭제를 요청하려면 플랫폼 관리자에게 문의하세요.

### A9. 내 게시물이 왜 "검토 대기 중"으로 표시되나요?

플랫폼의 현재 모더레이션 설정에 따라 새 게시물은 공개되기 전에 검토 과정을 거칠 수 있습니다. 모더레이터가 게시물을 검토하고 승인하거나 거부합니다.

### A10. 부적절한 콘텐츠를 어떻게 신고하나요?

게시물 또는 댓글의 세 점 메뉴(⋮)를 클릭하고 **신고**를 선택합니다. 이유를 선택하고 추가 세부 정보를 입력한 후 제출합니다. 모더레이터가 신고를 검토합니다.

### A11. 커뮤니티 게시물에 파일을 몇 개까지 첨부할 수 있나요?

게시물당 최대 **12개 파일** (이미지 및 문서)과 **4개 외부 링크**를 첨부할 수 있습니다. 개별 파일은 10MB 미만이어야 합니다.

### A12. 게시물에 업로드할 수 있는 파일 형식은 무엇인가요?

이미지: JPG, JPEG, PNG, WebP, GIF
문서: PDF, DOC, DOCX, PPT, PPTX, XLS, XLSX

## B섹션: 관리자 질문

### B1. 관리자 패널에 어떻게 접근하나요?

`{백엔드URL}/admin`으로 이동합니다. `admin` 역할을 가진 계정으로 로그인합니다.

### B2. 새 관리자 사용자를 어떻게 만드나요?

1. 일반적인 방법으로 사용자 계정을 등록하거나 생성합니다.
2. 관리자 패널에서 **사용자 → 사용자**로 이동합니다.
3. 사용자를 찾아 레코드를 편집합니다.
4. **역할**을 **관리자**로 변경하고 저장합니다.

> 또는 `php artisan tinker`를 사용하여 데이터베이스에서 역할을 수동으로 업데이트합니다.

### B3. 서비스 오픈 전 데모 콘텐츠를 어떻게 삭제하나요?

**관리자 패널 → 시스템 → 데모 데이터 정리**로 이동합니다. 데모 항목 목록을 검토하고 **데모 콘텐츠 삭제**를 클릭합니다. 이 작업은 되돌릴 수 없습니다.

### B4. 법적 페이지(개인정보처리방침/이용약관)를 어떻게 업데이트하나요?

**관리자 패널 → 시스템 → 법적 페이지 설정**으로 이동합니다. 세 가지 언어 모두에서 각 페이지의 콘텐츠를 편집합니다. 저장하면 변경사항이 즉시 적용됩니다.

### B5. 이메일 발송을 어떻게 설정하나요?

**관리자 패널 → 시스템 → 이메일 설정**으로 이동합니다. SMTP 자격증명을 입력하고 **테스트 이메일 발송**을 클릭하여 확인합니다. 설정이 즉시 적용됩니다.

### B6. 커뮤니티 게시물을 홈페이지에 추천하려면 어떻게 하나요?

관리자 패널에서 **커뮤니티 → 게시물**로 이동합니다. 게시물을 찾아 편집합니다. **추천** 체크박스를 선택하고 저장합니다. 게시물이 추천 게시물 섹션에 표시됩니다.

### B7. 운영 중에 로컬 스토리지에서 Azure로 전환할 수 있나요?

예, 하지만 주의가 필요합니다:
1. `FILESYSTEM_DISK=azure`로 업데이트하고 Azure 자격증명을 설정합니다.
2. 캐시 지우기: `php artisan optimize:clear`
3. 참고: 로컬 스토리지의 기존 파일은 자동으로 마이그레이션되지 않습니다. 미디어 스토리지 스캔 도구를 사용하여 불일치를 식별하세요.

### B8. 실패한 큐 작업을 어떻게 처리하나요?

백엔드 터미널에서:
```bash
php artisan queue:failed       # 실패한 작업 목록
php artisan queue:retry all    # 모든 실패한 작업 재시도
php artisan queue:flush        # 실패한 작업 초기화 (검토 후)
```

또는 관리자 패널의 인수인계 준비 페이지에서 요약을 확인하세요.

### B9. B2B 리드를 어떻게 내보내나요?

**관리자 패널 → B2B/리드 → B2B 리드**로 이동합니다. **내보내기** 버튼을 사용하여 모든 리드의 CSV 파일을 다운로드합니다.

### B10. 새 제품 카테고리를 어떻게 추가하나요?

**관리자 패널 → 스토어 → 제품 카테고리 → 새 카테고리**로 이동합니다. 세 가지 언어 모두에서 카테고리 이름을 입력하고 고유 슬러그를 설정합니다. 저장합니다.

### B11. 모든 주문을 어디서 볼 수 있나요?

**관리자 패널 → 스토어 → 주문**으로 이동합니다. 필터 옵션을 사용하여 상태, 날짜 또는 고객별로 좁힙니다.

### B12. 두 관리자가 동시에 같은 레코드를 편집할 수 있나요?

플랫폼에는 실시간 공동 편집 기능이 없습니다. 두 관리자가 동시에 같은 레코드를 편집하면 마지막 저장이 적용됩니다. 충돌을 피하기 위해 팀과 조율하세요.

## C섹션: 기술 담당자 질문

### C1. 데이터베이스 마이그레이션을 어떻게 실행하나요?

```bash
cd B2C_backend
php artisan migrate
```

상태 확인: `php artisan migrate:status`

### C2. 데모 데이터를 어떻게 시딩하나요?

```bash
php artisan db:seed
```

특정 시더 실행:
```bash
php artisan db:seed --class=ProductSeeder
```

### C3. i18n 키 커버리지를 어떻게 확인하나요?

```bash
cd B2C_frontend
npm run check:i18n
```

### C4. 애플리케이션 로그는 어디에 있나요?

`B2C_backend/storage/logs/laravel.log`

### C5. 새 번역 키를 어떻게 추가하나요?

1. 먼저 `B2C_frontend/messages/en.json`에 키를 추가합니다.
2. `ko.json`과 `zh.json`에 해당 번역을 추가합니다.
3. `npm run check:i18n`을 실행하여 커버리지를 확인합니다.
4. 프론트엔드를 재빌드합니다.

### C6. 스토리지 링크가 작동하는지 어떻게 확인하나요?

```bash
ls -la B2C_backend/public/storage
# 표시 예: storage -> /path/to/storage/app/public
```

링크가 없는 경우: `php artisan storage:link`

### C7. 테스트 스위트를 어떻게 실행하나요?

```bash
# 백엔드 (Laravel)
cd B2C_backend
php artisan test

# 프론트엔드 (단위 테스트가 설정된 경우)
cd B2C_frontend
npm run test

# 엔드투엔드 테스트 (Playwright, 프로젝트 루트에서)
cd terraf
npx playwright test
```

### C8. PHP 버전 요구 사항을 어떻게 확인하나요?

```bash
php --version
# 8.2 이상이어야 함

php -m | grep -E "pdo_mysql|mbstring|openssl"
# 필수 확장이 로드되었는지 확인
```

## D섹션: 클라이언트 / 사업주 질문

### D1. 플랫폼 이름을 OXP에서 변경할 수 있나요?

예. 백엔드 `.env` 파일의 `APP_NAME`을 업데이트하고, 프론트엔드 브랜딩 자산을 업데이트하고, 플랫폼 이름을 참조하는 이메일 템플릿을 업데이트하세요. 전체 리브랜딩을 위해 개발팀에 문의하세요.

### D2. 언어를 추가할 수 있나요?

예, 개발 작업이 필요합니다:
- `B2C_frontend/messages/{locale}.json`에 새 로케일 파일 추가
- i18n 라우팅 설정 업데이트
- 데이터베이스에 새 언어용 다국어 콘텐츠 컬럼 추가
- 새 언어 편집을 지원하도록 관리자 패널 업데이트

### D3. 결제 게이트웨이가 포함되어 있나요?

아니요. 현재 플랫폼은 주문과 결제 의도를 기록하지만 결제를 처리하지는 않습니다. Stripe, PayPal 또는 현지 공급자와 같은 결제 게이트웨이를 통합해야 합니다. 이는 개발 작업이 필요합니다.

### D4. 스토어를 국제 배송에 사용할 수 있나요?

현재 스토어는 뉴질랜드 (NZD 통화, NZ Post 배송)용으로 설정되어 있습니다. 국제 지원에는 통화 변환, 국제 운송업체 통합, 다중 통화 가격 책정이 필요합니다. 이는 개발 작업이 필요합니다.

### D5. 데이터는 누가 소유하나요?

모든 플랫폼 데이터는 귀하의 데이터베이스(MySQL)와 파일 스토리지(Azure Blob Storage 또는 로컬)에 저장됩니다. 모든 데이터의 완전한 소유권은 귀하에게 있습니다.

### D6. 플랫폼이 대규모 사용자와 주문을 처리할 수 있나요?

플랫폼은 중간 규모 운영에 적합하게 설계되었습니다. 높은 트래픽 시나리오의 경우 다음을 고려하세요:
- 캐시 및 큐에 Redis 사용 (데이터베이스 대신)
- 정적 자산을 위한 CDN 추가
- 높은 읽기 부하를 위한 데이터베이스 읽기 복제본
- 확장 권장 사항을 위해 개발팀에 문의

### D7. 플랫폼을 어떻게 백업하나요?

백업 절차는 [11-배포-및-유지보수.md](./11-배포-및-유지보수.md)를 참조하세요. 핵심 사항:
- MySQL 데이터베이스를 매일 백업
- Azure Blob Storage (또는 로컬 미디어 파일)를 정기적으로 백업
- `.env` 파일을 안전하게 백업 유지

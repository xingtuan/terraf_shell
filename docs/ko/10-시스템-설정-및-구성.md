# 10 — 시스템 설정 및 구성

## 개요

OXP는 두 가지 수준의 설정을 제공합니다:

1. **환경 수준 설정** — `.env` 파일에서 설정; 적용을 위해 서버 재시작이 필요; 데이터베이스, 스토리지, 메일 서버 등 근본적인 수준에서 애플리케이션에 영향을 미칩니다.
2. **관리자 수준 런타임 설정** — 관리자 패널 설정 페이지에서 설정; `app_settings` 데이터베이스 테이블에 저장; 재시작 없이 즉시 적용됩니다.

---

## 1. 환경 변수 참조 (`.env`)

`.env` 파일은 `B2C_backend/.env`에 위치합니다. 아래는 전체 참조 목록입니다.

### 1.1 애플리케이션

| 변수 | 기본값 | 설명 |
|---|---|---|
| `APP_NAME` | `OXP` | 애플리케이션 표시 이름 |
| `APP_ENV` | `local` | 환경: `local`, `production` |
| `APP_KEY` | (생성됨) | Laravel 암호화 키 — 반드시 설정 필요 |
| `APP_DEBUG` | `true` | 디버그 모드 — **프로덕션에서는 `false`로 설정** |
| `APP_URL` | `http://127.0.0.1:8000` | 백엔드 URL — 미디어 제공을 위해 정확해야 함 |
| `APP_LOCALE` | `en` | 기본 로케일 |
| `APP_FALLBACK_LOCALE` | `en` | 번역이 없을 때 대체 언어 |

### 1.2 데이터베이스

| 변수 | 기본값 | 설명 |
|---|---|---|
| `DB_CONNECTION` | `mysql` | 데이터베이스 드라이버 |
| `DB_HOST` | `127.0.0.1` | 데이터베이스 서버 호스트 |
| `DB_PORT` | `3306` | 데이터베이스 포트 |
| `DB_DATABASE` | `product_community` | 데이터베이스 이름 |
| `DB_USERNAME` | `root` | 데이터베이스 사용자명 |
| `DB_PASSWORD` | (없음) | 데이터베이스 비밀번호 |

### 1.3 스토리지

| 변수 | 기본값 | 설명 |
|---|---|---|
| `FILESYSTEM_DISK` | `azure` | 기본 스토리지 디스크: `azure` 또는 `public` |
| `COMMUNITY_UPLOAD_DISK` | `azure` | 커뮤니티 업로드용 디스크 |
| `AZURE_STORAGE_NAME` | (필수) | Azure 스토리지 계정 이름 |
| `AZURE_STORAGE_KEY` | (필수) | Azure 스토리지 계정 키 |
| `AZURE_STORAGE_CONTAINER` | `uploads` | Azure 컨테이너 이름 |
| `AZURE_STORAGE_URL` | (필수) | Azure Blob 엔드포인트 URL |
| `AZURE_STORAGE_USE_SAS_URLS` | `true` | 보안을 위한 시간 제한 SAS URL 사용 |
| `AZURE_STORAGE_SAS_URL_TTL_MINUTES` | `10080` | SAS URL 유효 기간 (7일) |

### 1.4 캐시 및 큐

## 2. 관리자 수준 런타임 설정

이 설정들은 관리자 패널에서 관리되며 `app_settings` 테이블에 저장됩니다. 해당하는 환경 기본값을 재정의합니다.

### 2.1 애플리케이션 설정

**위치**: 관리자 패널 → 시스템 → 앱 설정

설정 가능한 옵션:
- 애플리케이션 이름 재정의
- 기능 플래그 토글
- 공개 연락처 정보

### 2.2 이메일 설정

**위치**: 관리자 패널 → 시스템 → 이메일 설정

`.env` 파일을 편집하지 않고 메일 서버 설정을 변경할 수 있습니다:
- SMTP 호스트, 포트, 암호화
- SMTP 사용자명 및 비밀번호
- 발신 주소 및 이름
- 테스트 이메일 발송

> **참고**: 런타임 이메일 설정은 `.env` 이메일 설정보다 우선합니다.

*관련 코드: `app/Models/EmailSetting.php`, `app/Filament/Pages/EmailSettings.php`*

### 2.3 배송 설정

**위치**: 관리자 패널 → 시스템 → 배송 설정

모든 배송 요금을 설정합니다:
- 일반 및 빠른 배송 요금
- 농촌 추가 요금
- 무료 배송 기준 금액
- NZ Post를 위한 출발지 위치

### 2.4 세금 설정

**위치**: 관리자 패널 → 시스템 → 세금 설정

- GST 세율
- 세금 레이블 (장바구니 및 결제 페이지에 표시)
- 가격에 세금 포함 여부

### 2.5 스토리지 설정

**위치**: 관리자 패널 → 시스템 → 스토리지 설정

스토리지 백엔드를 전환합니다:
- `azure` — Azure Blob Storage (프로덕션 권장)
- `public` — 로컬 퍼블릭 스토리지 (storage:link 필요)

### 2.6 NZ Post 설정

**위치**: 관리자 패널 → 시스템 → NZ Post 설정

NZ Post 통합을 활성화 또는 비활성화하고 API 자격증명을 설정합니다.

### 2.7 커뮤니티 설정

**위치**: 관리자 패널 → 시스템 → 커뮤니티 설정

- 제출 정책 (자동 승인 / 승인 필요 / 제한됨)
- 신뢰할 수 있는 게시자 임계값

### 2.8 커뮤니티 모더레이션 설정

**위치**: 관리자 패널 → 시스템 → 커뮤니티 모더레이션 설정

- 민감어 필터링
- 콘텐츠 심사 규칙

### 2.9 법적 페이지 설정

**위치**: 관리자 패널 → 시스템 → 법적 페이지 설정

개인정보처리방침 및 이용약관 페이지의 내용을 편집합니다.

### 2.10 기능 플래그

**위치**: 관리자 패널 → 시스템 → 기능 플래그

코드 배포 없이 특정 플랫폼 기능을 활성화하거나 비활성화합니다.

## 3. 설정 보안

`app_settings` 테이블은 민감한 값에 **암호화**를 사용합니다. 즉:
- 설정으로 저장된 비밀번호 및 API 키는 저장 시 암호화됩니다.
- 데이터베이스에서 `app_settings` 테이블을 직접 편집하지 마세요 — 값을 읽을 수 없을 수 있습니다.
- 모든 설정 관리는 관리자 패널 인터페이스를 사용하세요.

`.env`의 `APP_KEY`는 암호화에 사용됩니다. 이 키가 변경되면 모든 암호화된 설정을 읽을 수 없게 됩니다.

*관련 코드: `app/Models/AppSetting.php`, `app/Models/AppSettingAuditLog.php`*

---

## 4. 설정 감사 로그

**위치**: 관리자 패널 → 시스템 → (설정 백업 페이지를 통해)

애플리케이션 설정에 대한 모든 변경은 `app_setting_audit_logs`에 기록됩니다:
- 어떤 설정이 변경되었는지
- 이전 및 새 값 (비밀 정보는 마스킹)
- 어떤 관리자가 변경했는지
- 타임스탬프

---

## 5. 설정 백업 및 가져오기

**위치**: 관리자 패널 → 시스템 → 설정 백업

**설정 내보내기:**
1. **설정 내보내기** 버튼을 클릭합니다.
2. 현재 관리자 설정이 포함된 JSON 파일이 다운로드됩니다.
3. 복구 또는 마이그레이션을 위해 이 파일을 안전하게 저장합니다.

`GET /admin/settings/export`를 통해서도 접근 가능합니다.

**설정 가져오기:**
1. **설정 백업 가져오기**로 이동합니다.
2. 이전에 내보낸 설정 JSON 파일을 업로드합니다.
3. 적용될 변경사항을 검토합니다.
4. 가져오기를 확인합니다.

> **주의**: 설정을 가져오면 현재 값이 덮어써집니다. 신뢰할 수 있는 소스에서만 가져오세요.

## 6. 프론트엔드 환경 변수

프론트엔드(`B2C_frontend`)는 별도의 `.env.local` 또는 `.env.production` 파일을 사용합니다:

| 변수 | 설명 |
|---|---|
| `NEXT_PUBLIC_API_URL` | 백엔드 API 기본 URL (예: `https://api.yoursite.com`) |
| `NEXT_PUBLIC_APP_URL` | 프론트엔드 앱 URL |

이 변수들은 프로덕션용 프론트엔드를 빌드하기 전에 올바르게 설정해야 합니다.

---

## 7. 설정 모범 사례

1. **`.env` 파일을 버전 관리에 커밋하지 마세요** — 비밀 정보가 포함되어 있습니다.
2. **프로덕션에서는 `APP_DEBUG=false`로 설정** — 디버그 모드는 민감한 오류 세부 정보를 노출합니다.
3. **프로덕션 환경에서는 `APP_ENV=production` 사용**.
4. **기본 비밀번호 변경** — 설정 후 즉시 관리자 계정 비밀번호를 업데이트하세요.
5. **서비스 오픈 전 이메일 설정** — 개발 환경에서 `MAIL_MAILER=log`는 실제 이메일을 발송하지 않습니다.
6. **실제 CORS 출처 설정** — `CORS_ALLOWED_ORIGINS`는 프로덕션에서 실제 프론트엔드 URL과 일치해야 합니다.
7. **큐 워커 사용** — `QUEUE_CONNECTION=database`로 설정하고 `php artisan queue:work`를 실행하여 안정적인 백그라운드 작업을 보장하세요.
8. **설정 정기 백업** — 변경 후 설정 내보내기 기능을 사용하세요.

# 03 — 관리자 매뉴얼

## 소개

OXP 관리자 패널은 **Filament 5**로 구축되어 있으며 백엔드 URL의 `/admin` 경로에서 접근할 수 있습니다. 이 매뉴얼은 관리자 패널의 모든 관리 기능을 다룹니다.

---

## 1. 관리자 로그인

1. `{백엔드URL}/admin`으로 이동합니다.
2. 관리자 이메일과 비밀번호를 입력합니다.
3. **로그인**을 클릭합니다.

> **기본 관리자 자격증명**: 초기 설정 시 `UserSeeder`에 의해 설정됩니다. 첫 로그인 후 즉시 변경하세요.

---

## 2. 대시보드

로그인 후 대시보드에서 플랫폼 활동의 빠른 개요를 확인할 수 있습니다:
- 최근 주문
- 커뮤니티 활동 요약
- 대기 중인 신고 수
- 사용자 가입 수
- 빠른 내비게이션 링크

---

## 3. 내비게이션 구조

좌측 사이드바:

| 그룹 | 내용 |
|---|---|
| **대시보드** | 개요 지표 |
| **커뮤니티** | 게시물, 댓글, 신고, 모더레이션 로그, 사용자 위반, 관리자 행동 로그 |
| **사용자** | 사용자 관리 |
| **스토어** | 주문, 제품, 옵션, 카테고리, 이미지, 속성, 재고, 장바구니, 주소 |
| **CMS** | 소재, 사양, 스토리 섹션, 활용 사례, 기사, 홈페이지 섹션, 아이디어 미디어 |
| **B2B/리드** | B2B 리드, 문의 |
| **이메일** | 이메일 템플릿, 이메일 이벤트, 이메일 로그 |
| **크라우드펀딩** | 펀딩 캠페인 |
| **커뮤니티 설정** | 카테고리, 태그 |
| **시스템** | 모든 설정 및 도구 |

---

## 4. 사용자 관리

**위치**: 사용자 → 사용자

### 4.1 사용자 역할

| 역할 | 코드 | 접근 권한 |
|---|---|---|
| 크리에이터 | `creator` | 일반 사용자 — 탐색, 게시, 쇼핑 |
| 모더레이터 | `moderator` | 크리에이터 모든 권한 + 모더레이션 |
| 관리자 | `admin` | 관리자 패널 완전 접근 |

### 4.2 계정 상태

| 상태 | 효과 |
|---|---|
| 활성 (active) | 정상 계정 운영 |
| 제한 (restricted) | 로그인 가능하지만 게시 또는 댓글 불가 |
| 정지 (suspended) | 로그인 불가 |
| 차단 (banned) | 영구 차단 — 계정 잠금 |

---

## 5. 제품 관리

**위치**: 스토어 → 제품

### 5.1 제품 생성

**새 제품**을 클릭하여 다음을 작성합니다:

**기본 정보**
- **이름** (필수, 다국어: 영어/한국어/중국어)
- **슬러그** — 이름에서 자동 생성, 고유해야 함
- **카테고리** — 기존 제품 카테고리에서 선택
- **상태** — 임시저장, 활성, 비활성, 보관
- **설명**, **특징**, **관리 방법**, **소재 장점** (다국어)

**가격**
- **판매 가격** — NZD 기준
- **정가** — 세일 표시용 원래 가격

**커머스 설정**
- **문의 전용** — 활성화 시 "장바구니 담기" 버튼이 문의 버튼으로 변경
- **소재 요청 활성화** — 사용자가 소재을 요청할 수 있음
- **추천 제품** — 추천 섹션에 표시

**SEO** — 제목 및 설명 (다국어)

### 5.2 제품 옵션

각 제품에는 여러 옵션이 있을 수 있습니다 (예: 크기, 색상).

| 필드 | 설명 |
|---|---|
| `sku` | 재고 관리 코드 — 옵션별 고유 식별자 |
| `stock_quantity` | 현재 사용 가능한 재고 |
| `weight_grams` | 배송비 계산을 위한 패키지 무게 |
| `barcode` | 바코드 (선택 사항) |

## 6. 주문 관리

**위치**: 스토어 → 주문

### 6.1 주문 상태

| 상태 | 의미 |
|---|---|
| 처리 대기 중 | 접수됨, 미확인 |
| 확인 완료 | 관리자 확인 |
| 처리 중 | 준비 중 |
| 배송 완료 | 발송 완료 |
| 배달 완료 | 수령 확인 |
| 취소됨 | 취소됨 |

---

## 7. 커뮤니티 게시물 관리

**위치**: 커뮤니티 → 게시물

### 7.1 게시물 상태

| 상태 | 의미 |
|---|---|
| 임시저장 | 미게시; 작성자만 볼 수 있음 |
| 검토 대기 중 | 모더레이션 승인 대기 |
| 게시됨 | 공개 표시 |
| 보관됨 | 공개 피드에서 숨겨짐 |
| 거부됨 | 모더레이터가 거부 |

---

## 8. B2B 리드 관리

**위치**: B2B/리드 → B2B 리드

### 8.1 리드 상태 파이프라인

| 상태 | 설명 |
|---|---|
| 신규 (New) | 문의 제출됨 |
| 연락 완료 (Contacted) | 첫 연락 완료 |
| 자격 확인 (Qualified) | 리드가 자격 기준 충족 |
| 협상 중 (Negotiating) | 적극적인 논의 중 |
| 전환 완료 (Converted) | 거래 성사 |
| 거부됨 (Rejected) | 리드 거부 |

**내보내기**: B2B 리드 목록에서 **내보내기** 버튼을 클릭하여 CSV 파일을 다운로드합니다.

## 10. 이메일 센터

**위치**: 이메일 → 이메일 템플릿 / 이메일 이벤트 / 이메일 로그

- **이메일 템플릿**: 트랜잭션 이메일 템플릿 보기 및 편집 (다국어 지원)
- **이메일 이벤트**: 이메일 발송을 트리거하는 이벤트 유형 설정
- **이메일 로그**: 발송된 모든 이메일 기록 검토

---

**위치**: 시스템 → [다양한 설정 페이지]

| 설정 페이지 | 기능 |
|---|---|
| 앱 설정 | 앱 이름, 기능 플래그 등 |
| 이메일 설정 | SMTP 서버 설정 |
| 스토리지 설정 | Azure/로컬 스토리지 전환 |
| 배송 설정 | 배송비 규칙 설정 |
| 세금 설정 | GST/세율 및 레이블 |

| 기능 플래그 | 런타임 기능 토글 |

---

## 12. 초기 콘텐츠 관리

**위치**: 시스템 → 초기 콘텐츠 정리

**실제 서비스 오픈 전:**
1. **시스템 → 초기 콘텐츠 정리**로 이동합니다.
2. 초기 항목 목록을 검토합니다.
3. **초기 콘텐츠 삭제**를 클릭합니다.

> **경고**: 이 작업은 되돌릴 수 없습니다. 삭제 전 실제 콘텐츠가 준비되었는지 확인하세요.

---

## 13. 인수인계 준비

**위치**: 시스템 → 인수인계 준비

클라이언트에게 플랫폼을 전달하거나 프로덕션에 출시하기 전에 이 페이지를 사용하여 시스템 준비 상태를 검토합니다.

상태 표시: **OK (정상)**, **Warning (경고)**, **Error (오류)**

인수인계 전에 모든 **Error** 항목을 해결하고 모든 **Warning** 항목을 검토하세요.

---

## 14. 일상 운영 모범 사례

1. **매일 모더레이션 큐 확인** — 미해결 신고를 신속하게 처리하세요.
2. **실패한 작업 모니터링** — 인수인계 준비 페이지에서 백그라운드 작업 실패를 확인하세요.
3. **주문 상태 업데이트** — 고객이 정확한 정보를 볼 수 있도록 주문 상태를 최신으로 유지하세요.
4. **B2B 리드 정기 검토** — 새 리드는 영업일 기준 1일 내 팔로업하세요.
5. **설정 백업** — 중요한 변경 후 설정 내보내기를 수행하세요.
6. **이메일 발송 테스트** — 메일 서버 변경 후 테스트 이메일을 전송하여 확인하세요.
7. **초기 콘텐츠 삭제 유지** — 실제 고객 활동 전에 초기 콘텐츠가 정리되었는지 확인하세요.
8. **스토리지 모니터링** — 미디어 스토리지 스캔 도구를 사용하여 불필요한 파일을 정기적으로 정리하세요.

# 07 — 콘텐츠 관리 매뉴얼

## 개요

이 매뉴얼은 OXP 관리자 패널을 통해 관리되는 모든 콘텐츠를 다룹니다. 소재 라이브러리, 기사, 홈페이지 섹션, 법적 페이지, 푸터 콘텐츠가 포함됩니다. 모든 콘텐츠는 영어, 한국어, 중국어 간체로 다국어 편집을 지원합니다.

---

## 1. 소재 라이브러리

### 1.1 목적

소재 라이브러리는 브랜드의 주요 지속 가능한 소재를 소개합니다. 웹사이트의 과학적·서사적 핵심으로, 각 소재의 사양, 스토리, 활용 사례, 인증 정보를 제공합니다.

**프론트엔드 위치**: `/material`

### 1.2 소재 필드

**위치**: 관리자 패널 → CMS → 소재

| 필드 | 유형 | 다국어 | 설명 |
|---|---|---|---|
| `name` | 텍스트 | 예 | 소재 이름 |
| `description` | 리치 텍스트 | 예 | 소재 주요 설명 |
| `story_cover_image` | 이미지 | 아니요 | 소재 스토리 섹션 커버 이미지 |
| `internal initial-content marker` | 불리언 | 아니요 | 초기/시더 콘텐츠 표시 |
| `seeded_at` | 타임스탬프 | 아니요 | 초기 콘텐츠 시딩 시각 |
| 인증 필드 | 텍스트 | 예 | 품질 및 규정 준수 인증 |

### 1.3 소재 사양

**위치**: 관리자 패널 → CMS → 소재 사양

각 소재의 기술 사양 테이블입니다. 각 사양 항목:
- 상위 소재 참조
- 사양 이름/레이블 (다국어)
- 사양 값 (다국어)
- 정렬 순서

### 1.4 소재 스토리 섹션

**위치**: 관리자 패널 → CMS → 스토리 섹션

소재의 이야기를 전달하는 서사 섹션입니다. 각 섹션:
- 상위 소재 참조
- 섹션 제목 (다국어)
- 섹션 내용 (다국어, 리치 텍스트)
- 섹션 이미지
- 정렬 순서

### 1.5 소재 활용 사례

**위치**: 관리자 패널 → CMS → 활용 사례

각 소재의 실제 활용 사례 소개입니다:
- 상위 소재 참조
- 활용 제목 (다국어)
- 활용 설명 (다국어)
- 활용 이미지
- 정렬 순서

*관련 코드: `app/Models/Material.php`, `app/Models/MaterialSpec.php`, `app/Models/MaterialStorySection.php`, `app/Models/MaterialApplication.php`*

## 2. 기사

### 2.1 목적

기사는 플랫폼의 지식 베이스이자 블로그를 구성합니다. 관리자가 작성하며 브랜드 소재, 지속 가능성, 업계 관련 주제를 다룹니다.

**프론트엔드 위치**: `/articles`

### 2.2 기사 필드

**위치**: 관리자 패널 → CMS → 기사

| 필드 | 유형 | 다국어 | 설명 |
|---|---|---|---|
| `title` | 텍스트 | 예 | 기사 제목 |
| `slug` | 텍스트 | 아니요 | URL 식별자 (제목에서 자동 생성) |
| `cover_image` | 이미지 | 아니요 | 기사 목록 및 상세 페이지 헤더 이미지 |
| `content_json` | 리치 텍스트 (JSON) | 아니요 | Tiptap JSON 형식의 기사 전체 내용 |
| `excerpt` | 텍스트 | 예 | 기사 목록에 표시되는 요약 |
| `reading_time` | 정수 | 아니요 | 예상 읽기 시간 (분) |
| `status` | 열거형 | 아니요 | 게시됨 또는 임시저장 |
| `published_at` | 타임스탬프 | 아니요 | 기사가 공개된 시각 |

### 2.3 기사 작성

1. **CMS → 기사 → 새 기사**로 이동합니다.
2. 세 가지 언어(영어/한국어/중국어)로 **제목**을 입력합니다.
3. **커버 이미지**를 업로드합니다.
4. 리치 텍스트 편집기를 사용하여 기사 내용을 작성합니다.
5. 각 언어로 간략한 **요약**을 추가합니다.
6. 준비가 되면 **상태**를 게시됨으로 설정합니다.
7. **저장**을 클릭합니다.

### 2.4 게시 및 비공개 처리

- 상태를 **게시됨**으로 변경하면 웹사이트에서 기사가 표시됩니다.
- 상태를 **임시저장**으로 변경하면 공개 표시에서 숨겨집니다.
- `published_at` 타임스탬프는 최초 게시 시 자동으로 설정됩니다.

*관련 코드: `app/Models/Article.php`, `app/Filament/Resources/ArticleResource.php`*

## 3. 홈페이지 섹션

### 3.1 목적

홈페이지는 관리자 패널에서 관리되는 설정 가능한 섹션으로 구성됩니다. 코드 변경 없이 홈페이지 레이아웃과 콘텐츠를 업데이트할 수 있습니다.

**프론트엔드 위치**: `/` (홈페이지)

### 3.2 홈페이지 섹션 필드

**위치**: 관리자 패널 → CMS → 홈페이지 섹션

| 필드 | 유형 | 다국어 | 설명 |
|---|---|---|---|
| `section_type` | 텍스트 | 아니요 | 이 섹션을 렌더링할 프론트엔드 컴포넌트 식별자 |
| `title` | 텍스트 | 예 | 섹션 제목 |
| `subtitle` | 텍스트 | 예 | 섹션 보조 텍스트 |
| `content_json` | JSON | 아니요 | 구조화된 섹션 콘텐츠 (레이아웃에 따라 다름) |
| `background_image` | 이미지 | 아니요 | 섹션 배경 이미지 |
| `sort_order` | 정수 | 아니요 | 페이지에서의 표시 순서 제어 |
| `is_published` | 불리언 | 아니요 | 홈페이지에 이 섹션이 표시될지 여부 |

### 3.3 홈페이지 콘텐츠 관리

1. **CMS → 홈페이지 섹션**으로 이동합니다.
2. 편집할 섹션을 클릭합니다.
3. 필요에 따라 제목, 부제목, 이미지 또는 콘텐츠를 업데이트합니다.
4. `sort_order`를 조정하여 페이지에서 섹션 순서를 변경합니다.
5. `is_published`를 토글하여 개별 섹션을 표시하거나 숨깁니다.
6. 변경사항을 저장합니다.

> **참고**: 프론트엔드는 `section_type` 값을 특정 React 컴포넌트에 매핑합니다. 기존 섹션의 `section_type`을 변경하면 표시 문제가 발생할 수 있습니다. 섹션 유형을 변경하기 전에 개발팀에 문의하세요.

*관련 코드: `app/Models/HomeSection.php`, `app/Filament/Resources/HomeSectionResource.php`*

## 4. 법적 페이지

### 4.1 목적

개인정보처리방침 및 이용약관 페이지는 관리자 패널에서 직접 관리되며 API를 통해 동적으로 제공됩니다.

**프론트엔드 위치**: `/privacy`, `/terms`

### 4.2 법적 페이지 편집

**위치**: 관리자 패널 → 시스템 → 법적 페이지 설정

법적 페이지 편집기 제공 기능:
- **개인정보처리방침** 및 **이용약관** 별도 편집기
- **영어**, **한국어**, **중국어 간체** 콘텐츠 편집
- 리치 텍스트 편집 지원
- 각 페이지의 **"최종 업데이트" 날짜** 필드

### 4.3 권장 절차

1. 담당 법률 전문가에게 법적 문서를 검토받습니다.
2. 승인된 텍스트를 세 가지 언어 모두로 입력합니다.
3. **"최종 업데이트"** 날짜를 현재 날짜로 설정합니다.
4. 저장 — 변경사항이 즉시 웹사이트에 반영됩니다.

> **중요**: 프로덕션 환경에서 임시 또는 초안 법적 문서를 사용하지 마세요. 법률 검토가 완료된 내용만 게시하세요.

*관련 코드: `app/Filament/Pages/LegalPageSettings.php`, `app/Http/Controllers/Api/LegalPageController.php`*

## 5. 푸터 콘텐츠

푸터에는 내비게이션 링크와 연락처 정보가 포함됩니다. 현재 구현에서 푸터 링크는 홈페이지 섹션 시스템을 통해 관리되거나 프론트엔드 푸터 컴포넌트에 하드코딩되어 있습니다.

**프론트엔드 컴포넌트**: `B2C_frontend/src/components/footer.tsx`

푸터 링크나 콘텐츠를 업데이트하려면 개발팀에 문의하세요.

## 6. 다국어 콘텐츠 편집

콘텐츠가 풍부한 모든 필드는 세 가지 언어를 지원합니다: 영어(`en`), 한국어(`ko`), 중국어 간체(`zh`).

### 6.1 관리자 패널에서의 편집

제품, 소재, 기사 또는 홈페이지 섹션을 편집할 때 관리자 패널에는 각 언어별 별도 입력 필드가 표시됩니다:
- `name_en`, `name_ko`, `name_zh`
- `description_en`, `description_ko`, `description_zh`
- 기타 등등

완전한 다국어 경험을 위해 세 가지 언어 버전을 모두 작성하세요.

### 6.2 대체(Fallback) 동작

현재 방문자 언어로 번역이 없는 경우 시스템은 **영어 버전** 콘텐츠로 대체됩니다.

### 6.3 프론트엔드 번역 파일

UI 문자열 (버튼 레이블, 오류 메시지, 섹션 제목)은 다음 파일에 저장됩니다:
- `B2C_frontend/messages/en.json`
- `B2C_frontend/messages/ko.json`
- `B2C_frontend/messages/zh.json`

이 파일들을 업데이트하려면 개발자가 변경하고 새 프론트엔드 빌드를 배포해야 합니다.

## 7. 이미지 및 미디어

### 7.1 관리자 패널에서 이미지 업로드

이미지는 해당 관리자 리소스(제품, 소재, 기사, 홈페이지 섹션)에서 직접 업로드합니다. 이미지 업로드 필드를 클릭하면:
1. 컴퓨터에서 이미지 파일을 선택합니다.
2. 이미지가 Azure Blob Storage(또는 개발 환경이면 로컬 디스크)에 업로드됩니다.
3. URL 또는 스토리지 경로가 레코드에 저장됩니다.

**지원 형식**: JPG, JPEG, PNG, WebP, GIF

### 7.2 제품 이미지

**위치**: 스토어 → 제품 이미지

제품 이미지는 별도로 관리됩니다:
- 제품당 하나 이상의 이미지 업로드
- `sort_order` 설정으로 표시 순서 제어
- 이미지 하나를 **기본 이미지**로 지정하여 메인 표시 이미지로 설정

### 7.3 미디어 스토리지 관리

**위치**: 관리자 패널 → 시스템 → 미디어 스토리지 스캔

이 도구는 스토리지를 스캔하여 다음을 식별합니다:
- 어떤 레코드에서도 참조되지 않는 파일 (고아 파일)
- 누락된 파일을 참조하는 레코드

스토리지를 깔끔하게 유지하기 위해 주기적으로 사용하세요.

## 8. SEO 필드

제품과 기사에는 SEO 메타데이터 필드가 포함됩니다:
- `seo_title` — 브라우저 탭 제목 및 `<title>` 태그로 사용
- `seo_description` — `<meta name="description">` 태그로 사용

이 필드들은 다국어를 지원하며, 각 언어별로 입력하면 다양한 시장에서의 검색 엔진 가시성을 최적화할 수 있습니다.

## 9. 초기 콘텐츠 관리

초기 콘텐츠는 초기 설정 시 `php artisan db:seed`로 시딩됩니다. 모든 초기 항목에는 `internal initial-content marker = true`와 `seeded_at` 타임스탬프가 표시됩니다.

**서비스 오픈 전**, 모든 초기 콘텐츠를 삭제하세요:
1. **관리자 패널 → 시스템 → 초기 콘텐츠 정리**로 이동합니다.
2. 초기 항목 목록을 검토합니다.
3. **초기 콘텐츠 삭제**를 클릭합니다.

> **경고**: 이 작업은 되돌릴 수 없습니다. 초기 콘텐츠를 삭제하기 전에 실제 콘텐츠가 추가되었는지 확인하세요.

*관련 코드: `app/Filament/Pages/content management resources`*

## 10. 콘텐츠 관리 모범 사례

1. **게시 전 세 가지 언어 버전을 모두 작성**하세요.
2. **영어로 의미 있는 슬러그**를 사용하세요 (슬러그는 언어 간 공유됩니다).
3. **업로드 전 이미지를 최적화**하세요 — 파일 크기를 적절하게 유지하세요 (가능하면 이미지당 2MB 이하).
4. **법적 페이지는 게시 전 법률 전문가의 검토**를 받으세요.
5. **변경 후 홈페이지 섹션을 테스트**하여 올바르게 렌더링되는지 확인하세요.
6. 더 이상 관련 없는 기사나 게시물은 **삭제보다는 보관**하세요 — 삭제는 영구적입니다.
7. 주요 콘텐츠 구조 변경 후에는 설정 백업 도구를 사용하여 **설정을 백업**하세요.

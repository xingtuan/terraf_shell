# 08 — 주문 및 고객 관리

## 개요

이 매뉴얼은 관리자 관점에서의 주문 처리, 고객 계정 관리, 주소 관리를 다룹니다.

---

## 1. 주문 관리

### 1.1 주문 목록

**위치**: 관리자 패널 → 스토어 → 주문

주문 목록에는 모든 주문이 다음 정보와 함께 표시됩니다:
- 주문 번호 (형식: `OXP-XXXXXX`)
- 고객 (회원: 이름 및 이메일; 비회원: 이메일)
- 주문 합계
- 주문 상태
- 결제 상태
- 주문 날짜

**필터 옵션:**
- 주문 상태별 필터 (처리 대기, 확인 완료, 처리 중, 배송 완료, 배달 완료, 취소됨)
- 결제 상태별 필터 (미결제, 결제 완료, 환불됨, 실패)
- 주문 번호 또는 고객 이메일로 검색
- 날짜, 합계 또는 상태별 정렬

### 1.2 주문 상세 보기

주문을 클릭하면 전체 주문 상세 정보가 열립니다:

**주문 정보**
- 주문 번호
- 주문 일시
- 고객 유형 (비회원 또는 회원)
- 고객 이메일/사용자 계정 링크

**주문 항목**
- 제품 옵션 이름 및 SKU
- 주문 수량
- 주문 시점의 단가 (가격 스냅샷)
- 항목 소계

**재무 요약**
- 소계
- 배송비
- 세금 금액 (GST)
- 최종 합계
- 할인 (해당하는 경우)

**배송 정보**
- 전체 배송 주소 (주문 시점의 스냅샷)
- 선택한 배송 방법
- 배송 견적 스냅샷 (NZ Post 사용 시 운송업체 및 서비스 수준 포함)

**상태 정보**
- 현재 주문 상태
- 상태 타임스탬프 (confirmed_at, processing_at, shipped_at, delivered_at, cancelled_at)
- 현재 결제 상태

*관련 코드: `app/Models/Order.php`, `app/Filament/Resources/OrderResource.php`*

### 1.3 주문 상태 업데이트

주문 상세 페이지에서 **편집**을 클릭하여 수정합니다:
- **주문 상태** — 드롭다운에서 새 상태 선택
- **결제 상태** — 결제 확인 후 업데이트
- **결제 참조 번호** — 결제 처리업체의 참조 번호 기록
- **배송 추적 정보** (해당하는 경우 배송 메모에 추가)

**상태 흐름 권장 사항:**
1. 새 주문은 **처리 대기**로 도착합니다 — 빠르게 검토하고 확인하세요.
2. 수령을 확인하려면 **확인 완료**로 설정합니다.
3. 주문 준비 중에는 **처리 중**으로 설정합니다.
4. 패키지 발송 시 **배송 완료**로 설정합니다.
5. 배송 확인 후 또는 적절한 배송 기간 후 **배달 완료**로 설정합니다.

### 1.4 주문 취소

주문을 취소하려면:
1. 주문을 엽니다.
2. 편집하여 상태를 **취소됨**으로 변경합니다.
3. 필요한 경우 메모를 추가합니다.
4. 저장합니다.

> **참고**: 취소 시 자동으로 환불이 처리되지 않습니다. 결제 처리업체를 통해 수동으로 환불을 처리하세요.

## 2. 주문 상태 참고표

| 상태 | 사용 시점 | 고객에게 표시 |
|---|---|---|
| **처리 대기** | 주문 접수, 미확인 | "주문 접수됨" |
| **확인 완료** | 관리자 확인 완료 | "주문 확인됨" |
| **처리 중** | 제품 준비 중 | "준비 중" |
| **배송 완료** | 패키지 발송됨 | "배송됨" |
| **배달 완료** | 배달 확인됨 | "배달됨" |
| **취소됨** | 주문 취소됨 | "취소됨" |

## 3. 결제 상태 참고표

| 상태 | 사용 시점 |
|---|---|
| **미결제** | 주문 생성 시 기본값; 결제 미수령 |
| **결제 완료** | 관리자가 결제 확인 |
| **환불됨** | 전체 또는 부분 환불 처리 |
| **실패** | 결제 시도 실패 |

> **알림**: 플랫폼에는 자동 결제 게이트웨이가 포함되어 있지 않습니다. 모든 결제 상태 변경은 수동으로 처리해야 합니다.

## 4. 비회원 주문 조회

비회원 고객은 계정 없이 주문을 조회할 수 있습니다:

- **프론트엔드**: `/store/orders`
- **필요 정보**: 주문 번호(OXP-XXXXXX)와 결제 시 사용한 이메일 주소
- 두 가지 정보가 일치하면 주문 상세 내용이 반환됩니다.

관리자는 주문 목록에서 모든 비회원 주문을 볼 수 있습니다. 비회원 주문은 사용자 계정 없이 비회원 이메일만 있습니다.

*관련 코드: `app/Http/Controllers/Api/OrderController.php::lookupGuest()`*

## 5. 고객 계정 관리

### 5.1 고객 계정 보기

**위치**: 관리자 패널 → 사용자 → 사용자

역할별로 필터링하여 일반 고객(역할: `creator`)을 확인합니다. 각 고객 프로필에는 다음이 표시됩니다:
- 가입 날짜
- 계정 상태
- 프로필 정보
- 연결된 주문
- 커뮤니티 활동 (게시물, 댓글)

### 5.2 계정 상태 관리

고객이 계정 문제를 신고하는 경우 관리자는:
- 잘못 정지된 계정의 상태를 **활성**으로 재설정
- 비정상적인 활동이 있는 계정을 **정지** (임시)
- 정책 위반 계정을 **차단** (영구)
- 게시 기능을 막으면서 쇼핑은 유지하도록 계정 **제한**

> **중요**: 계정 모더레이션을 신중하게 사용하세요. 가능한 경우 조치 전에 고객에게 알리세요.

## 6. 주소 관리

### 6.1 관리자 주소 보기

**위치**: 관리자 패널 → 스토어 → 주소

관리자는 저장된 모든 고객 주소를 볼 수 있습니다. 이는 주로 지원 목적입니다 (예: 고객이 결제 문제를 보고할 때 기본 주소 확인).

각 주소 레코드에는 다음이 표시됩니다:
- 연결된 사용자
- 전체 주소 상세 정보
- 기본 주소 여부
- 생성 날짜

### 6.2 고객 자체 주소 관리

고객은 다음에서 자신의 주소를 직접 관리합니다:
- **프론트엔드**: 계정 → 주소
- 주소를 추가, 편집, 삭제하고 기본 주소를 설정할 수 있습니다.

---

## 7. 장바구니 관리 (관리자 보기)

**위치**: 관리자 패널 → 스토어 → 장바구니

관리자는 디버깅 및 지원 목적으로 활성 장바구니를 볼 수 있습니다:
- 고객의 장바구니 항목
- 수량 및 제품 옵션
- 장바구니 생성 및 업데이트 타임스탬프

이 보기는 관리자에게 읽기 전용입니다. 장바구니는 고객과 API에서 관리됩니다.

---

## 8. 주문 분석

관리자 패널은 다음을 통해 기본 주문 분석을 제공합니다:
- `GET /api/admin/analytics/overview` — 플랫폼 전체 참여도 및 거래 요약
- 메인 관리자 대시보드의 대시보드 위젯

상세한 판매 보고를 위해서는 관리자 패널의 필터 및 내보내기 기능을 사용하거나 데이터베이스를 직접 조회하세요.

---

## 9. 재고 조정

주문 처리 후 재고를 수동으로 업데이트합니다:

1. **관리자 패널 → 스토어 → 재고**로 이동합니다.
2. 해당 제품 옵션을 찾습니다.
3. **편집**을 클릭합니다.
4. `stock_quantity` 필드를 업데이트합니다.
5. 재고 조정 메모를 기록합니다.

> **권장 사항**: 재고 수준을 정확하게 유지하기 위해 일별 또는 주별 정기 재고 조정 프로세스를 수립하세요.

*관련 코드: `app/Models/InventoryAdjustment.php`, `app/Filament/Resources/InventoryResource.php`*

## 10. 일반적인 주문 처리 시나리오

### 10.1 주문 접수 후 결제 미확인

1. 고객에게 결제 시도 여부를 확인합니다.
2. 결제 처리업체에서 거래 내역을 확인합니다.
3. 결제 확인 시: `payment_status`를 결제 완료로 업데이트합니다.
4. 결제 실패 시: `payment_status`를 실패로 업데이트하고 고객에게 재시도를 요청합니다.

### 10.2 고객 취소 요청

1. 주문이 **처리 대기** 또는 **확인 완료** 상태이면: 문제없이 취소 가능합니다.
2. 주문이 **처리 중** 상태이면: 아직 중단할 수 있는지 평가합니다.
3. 주문이 **배송 완료** 상태이면: 물건을 회수할 수 없으므로 배달 후 반품/환불 처리합니다.

### 10.3 잘못된 주소 입력

1. 주문 상태 확인 — **처리 대기** 또는 **확인 완료** 상태이면 주소를 업데이트할 수 있을 수 있습니다.
2. 고객에게 연락하여 정확한 주소를 확인합니다.
3. 참고: 주문의 배송 주소는 스냅샷이므로 고객이 저장된 주소를 업데이트해도 자동으로 변경되지 않습니다.

### 10.4 주문 후 품절 상품

1. 고객에게 연락하여 지연 또는 대체 옵션을 알립니다.
2. 상황을 반영하도록 주문 상태를 업데이트합니다.
3. 고객이 원하는 경우 환불을 처리합니다.

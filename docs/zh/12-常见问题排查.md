# 12 — 常见问题排查

## 概述

本指南涵盖 OXP 平台开发、部署和日常运营过程中常见问题的解决方案。

---

## 1. 登录问题

### 1.1 无法登录——凭据无效

**症状**：登录时提示"凭据无效"。

**解决方案**：
1. 确认邮箱和密码正确（区分大小写）。
2. 确认用户账号存在于数据库。
3. 检查账号状态是否为 `suspended`（暂停）或 `banned`（封禁）。
4. 使用 seeder 默认管理员凭据时，查看 `database/seeders/UserSeeder.php` 确认默认值。

### 1.2 管理员无法访问后台

**症状**：访问 `/admin` 时被重定向至登录页或显示"未授权"。

**解决方案**：
1. 确认用户在 `users` 表中 `role = admin`。
2. 确认 `.env` 中的 `APP_URL` 与访问地址一致。
3. 清除缓存：`php artisan optimize:clear`

### 1.3 密码重置邮件未收到

**解决方案**：
1. 确认生产环境 `MAIL_MAILER` 不是 `log` 或 `null`。
2. 在管理后台 **系统 → 邮件设置 → 发送测试邮件** 验证邮件配置。
3. 查看 `storage/logs/laravel.log` 中的邮件错误。
4. 检查垃圾邮件文件夹。

---

## 2. 商品保存/管理后台问题

### 2.1 商品无法保存

**解决方案**：
1. 检查表单中的验证错误（必填字段、格式问题）。
2. 确认商品 `slug` 唯一（重复别名会验证失败）。
3. 查看 `storage/logs/laravel.log` 中的详细错误。

### 2.2 管理后台图片上传失败

**解决方案**：
1. 检查存储磁盘配置（`FILESYSTEM_DISK`）是否正确。
2. Azure 模式：验证 `AZURE_STORAGE_NAME`、`AZURE_STORAGE_KEY` 和容器名称。
3. 本地模式：运行 `php artisan storage:link` 并确认符号链接存在。
4. 检查文件大小是否超限。
5. 检查 `php.ini` 中的 `upload_max_filesize` 和 `post_max_size`。

---

## 3. 购物车和结账问题

### 3.1 购物车商品消失

**解决方案**：
1. 游客购物车依赖浏览器会话——确保启用 Cookies。
2. 登录用户：检查数据库中的 `carts` 和 `cart_items` 表。
3. 确认 `SANCTUM_STATEFUL_DOMAINS` 包含前台域名。
4. 通过浏览器开发者工具的 Network 标签检查 API 错误。

### 3.2 运费选项未显示

**解决方案**：
1. `NZPOST_ENABLED=true` 时：验证新西兰邮政 API 凭据有效。
2. 确认 `STORE_STANDARD_SHIPPING_RATE` 已设置固定费率值。
3. 查看 `POST /api/store/shipping-options` API 响应。

---

## 4. 邮件发送问题

### 4.1 邮件未发出

**开发环境**：查看 `storage/logs/laravel.log`——`MAIL_MAILER=log` 时邮件写入日志。

**生产环境**：
1. 在管理后台 **系统 → 邮件设置** 中验证 SMTP 凭据。
2. 点击**发送测试邮件**测试。
3. 检查垃圾邮件文件夹。
4. 确认发件地址已获得邮件服务提供商授权（SPF/DKIM 记录）。

---

## 5. 文件上传问题

### 5.1 社区帖子附件上传失败

**解决方案**：
1. 检查文件类型是否在 `IDEA_MEDIA_ALLOWED_EXTENSIONS` 中。
2. 检查文件大小是否超过 `IDEA_MEDIA_MAX_FILE_SIZE_KB`。
3. 确认存储磁盘配置正确且可访问。
4. 检查 `ALLOW_GUEST_UPLOAD=false` 是否阻止了未登录用户上传。
5. 查看 `storage/logs/laravel.log` 详细错误。

---

## 6. 翻译/多语言问题

### 6.1 文本显示为翻译键而非实际内容

**症状**：用户看到 `community.posts.title` 而非翻译文本。

**解决方案**：
1. 检查 `messages/{locale}.json` 中是否存在该翻译键。
2. 在前端目录运行 `npm run check:i18n` 查找缺失的翻译。
3. 更新翻译文件后重新构建：`npm run build`。

### 6.2 内容未按所选语言显示

**症状**：选择中文后产品或材料仍显示英文。

**解决方案**：
1. 检查管理后台中该内容是否已填写多语言字段。
2. 若非英文字段为空，系统自动回退至英文——这是预期行为。
3. 在管理后台补充对应语言的内容。

## 7. 缓存问题

### 7.1 保存后变更未生效

```bash
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
```

### 7.2 修改 .env 后出现 500 错误

```bash
php artisan optimize:clear
php artisan optimize
```

## 8. 数据库迁移问题

### 8.1 迁移失败：表已存在

**解决方案**：
1. 检查迁移状态：`php artisan migrate:status`
2. 如迁移部分完成：`php artisan migrate:rollback` 回滚最后一批。
3. 非必要不手动删除 `migrations` 表中的记录。

### 8.2 迁移失败：外键约束错误

**解决方案**：
1. 确保按照正确顺序运行迁移——外键迁移依赖父表先存在。
2. 检查迁移文件中的表名和列名是否正确。

---

## 9. 前端构建问题

### 9.1 构建时类型错误

```bash
# 检查类型
npm run typecheck

# 检查 i18n 键
npm run check:i18n
```

修复所有报错后重新构建。

### 9.2 前端无法访问后端 API

**症状**：所有 API 调用失败或浏览器控制台显示 CORS 错误。

**解决方案**：
1. 确认前端 `.env` 中的 `NEXT_PUBLIC_API_URL` 与后端地址一致。
2. 检查后端 `CORS_ALLOWED_ORIGINS` 是否包含前端源。
3. 确认后端运行正常且可访问。
4. 检查 HTTPS/HTTP 是否不一致。

---

## 10. 500 服务器错误

### 10.1 排查通用 500 错误

1. 查看 `storage/logs/laravel.log` 获取完整错误跟踪。
2. **开发环境**：临时设置 `APP_DEBUG=true` 在 API 响应中查看详细错误。
3. **绝不**在生产环境设置 `APP_DEBUG=true`。

### 10.2 部署后出现 500 错误

```bash
php artisan optimize:clear
php artisan optimize
php artisan migrate:status
chmod -R 775 storage
chmod -R 775 bootstrap/cache
```

---

## 11. 队列/后台任务问题

### 11.1 通知未发送

1. 检查队列工作者是否运行：`ps aux | grep queue:work`
2. 检查失败任务：`php artisan queue:failed`
3. 开发环境：设置 `QUEUE_CONNECTION=sync` 立即执行任务。

### 11.2 失败任务持续累积

```bash
# 查看失败任务
php artisan queue:failed

# 重试全部失败任务
php artisan queue:retry all

# 清除失败任务（调查后执行）
php artisan queue:flush
```

---

## 12. 权限问题

### 12.1 存储写入权限被拒

```bash
chmod -R 775 storage
chmod -R 775 bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
```

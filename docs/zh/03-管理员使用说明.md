# 03 — 管理员使用说明

## 简介

OXP 管理后台基于 **Filament 5** 构建，通过后端 URL 的 `/admin` 路径访问。本手册涵盖管理后台中所有可用的管理功能。

---

## 1. 管理员登录

1. 访问 `{后端URL}/admin`
2. 输入管理员邮箱和密码。
3. 点击**登录**。

> **默认管理员凭据**：由 `UserSeeder` 在初始化时设置。首次登录后请立即更改密码。

如无法登录：
- 确认用户在数据库中具有 `admin` 角色。
- 确认后端服务已运行。
- 检查 `.env` 中的 `APP_URL` 是否与访问地址一致。

---

## 2. 控制面板（仪表盘）

登录后，仪表盘提供平台活动的快速概览：
- 最近订单
- 社区动态摘要
- 待处理举报数量
- 用户注册数
- 快捷导航链接

---

## 4. 用户管理

**位置**：用户 → 用户

### 4.1 查看用户

用户列表显示所有注册用户，包含：
- 用户名和邮箱
- 角色（创作者/版主/管理员）
- 账号状态（正常/受限/暂停/封禁）
- 注册日期

使用搜索栏按姓名或邮箱查找用户，使用筛选器按角色或状态过滤。

### 4.2 编辑用户

点击任意用户行打开详情页，可以：
- 查看用户完整信息
- 更改用户**角色**
- 更改用户**账号状态**
- 查看关联帖子、订单和举报记录

### 4.3 角色说明

| 角色 | 代码 | 权限 |
|---|---|---|
| 创作者 | `creator` | 普通用户——浏览、发帖、购物 |
| 版主 | `moderator` | 创作者全部权限 + 审核操作 |
| 管理员 | `admin` | 完整后台访问权限 |

> **注意**：仅向可信人员分配 `admin` 角色，管理员拥有所有平台数据的无限制访问权限。

### 4.4 账号状态说明

| 状态 | 效果 |
|---|---|
| 正常 (active) | 账号正常运行 |
| 受限 (restricted) | 可登录但无法发帖或评论 |
| 暂停 (suspended) | 无法登录 |
| 封禁 (banned) | 永久封禁，账号锁定 |

## 5. 商品管理

**位置**：商城 → 产品

### 5.1 创建商品

点击**新建产品**，填写以下信息：

**基本信息**
- **名称**（必填，支持多语言：英/韩/中）
- **别名（Slug）** — 从名称自动生成，必须唯一
- **分类** — 从现有商品分类中选择
- **状态** — 草稿、上架、下架或存档
- **描述**（多语言）
- **特性**（多语言）
- **护理说明**（多语言）
- **材料优点**（多语言）

**价格**
- **售价** — 以新西兰元（NZD）为单位
- **原价** — 用于显示促销对比价

**商务设置**
- **仅询价** — 启用后"加入购物车"按钮替换为询盘按钮
- **启用材料申请** — 允许用户申请评估材料
- **精选商品** — 标记为精选商品

**SEO**
- **SEO 标题**（多语言）
- **SEO 描述**（多语言）

**商品 FAQ** — 添加问答项（多语言）

点击**保存**完成创建。

### 5.2 商品变体

**位置**：商城 → 产品变体

每个商品可有多个变体（如不同尺寸或颜色）。每个变体包含：
- **SKU** — 唯一标识符
- **所属商品** — 父商品
- **库存数量**
- **重量（克）**
- **条形码** — 可选

### 5.3 商品分类

**位置**：商城 → 产品分类

- **名称**（多语言：英/韩/中）
- **别名（Slug）** — 唯一 URL 标识符
- **父分类** — 用于层级分类

### 5.4 库存管理

**位置**：商城 → 库存

查看和调整商品变体的库存数量：
- 每个变体的当前库存量
- 记录库存调整（含备注）
- 查看调整历史

## 7. 社区帖子管理

**位置**：社区 → 帖子

### 7.1 帖子状态说明

| 状态 | 含义 |
|---|---|
| 草稿 (Draft) | 未发布，仅作者可见 |
| 待审核 (Pending Review) | 等待审核批准 |
| 已发布 (Published) | 公开可见 |
| 已存档 (Archived) | 从公开动态隐藏，内容保留 |
| 已拒绝 (Rejected) | 被版主拒绝，不公开显示 |

### 7.2 分类和标签管理

**位置**：社区设置 → 分类 / 标签

管理社区帖子使用的分类体系：
- 创建、编辑和删除分类与标签
- 支持多语言名称（英/韩/中）

---

## 8. 举报与内容审核管理

**位置**：社区 → 举报

详细内容请参阅 [09-审核举报与风控管理.md](./09-审核举报与风控管理.md)。

可执行的操作：
- **驳回** — 无需操作关闭举报
- **隐藏目标** — 从公开动态隐藏被举报内容
- **警告用户** — 向内容作者发送警告通知
- **限制用户** — 禁止内容作者发帖
- **封禁用户** — 永久封禁内容作者

## 9. B2B 线索管理

**位置**：B2B/线索 → B2B 线索

### 9.1 线索状态流程

| 状态 | 说明 |
|---|---|
| 新线索 (New) | 询盘刚提交 |
| 已联系 (Contacted) | 已完成首次沟通 |
| 已认证 (Qualified) | 线索符合资质标准 |
| 洽谈中 (Negotiating) | 正在积极沟通 |
| 已成交 (Converted) | 合作达成 |
| 已拒绝 (Rejected) | 线索不符合要求 |

### 9.2 导出线索

点击 B2B 线索列表中的**导出**按钮，下载 CSV 文件，用于 CRM 导入或销售报表。

## 13. 初始内容管理

**位置**：系统 → 初始内容清除

**正式运营前：**
1. 前往**系统 → 初始内容清除**。
2. 查看标记为初始内容的项目。
3. 点击**删除初始内容**。

> **警告**：此操作不可撤销。删除前请确保真实内容已准备就绪。

---

## 14. 交付就绪检查

**位置**：系统 → 交付就绪

正式交付前，使用此页面检查系统状态：
- 应用名称和 URL 配置
- 数据库连接状态
- 存储磁盘状态
- 邮件配置
- 队列模式
- 管理员账号
- 初始内容状态
- 失败任务
- PHP 和 Laravel 版本

状态标识：**OK（正常）**、**Warning（警告）**、**Error（错误）**。

交付前解决所有 **Error** 项，审查所有 **Warning** 项。

---

## 15. 日常运营最佳实践

1. **每日检查审核队列** — 及时处理待处理的举报。
2. **监控失败任务** — 在交付就绪页面检查后台失败任务。
3. **及时更新订单状态** — 保持订单状态准确，让客户看到最新进展。
4. **定期跟进 B2B 线索** — 新线索建议在1个工作日内回复。
5. **备份设置** — 每次重大配置变更后导出设置。
6. **测试邮件发送** — 更改邮件服务器后，发送测试邮件验证正常。
7. **清除初始内容** — 真实客户使用前确保已删除初始内容。
8. **监控存储** — 使用媒体存储扫描工具定期清理孤立文件。

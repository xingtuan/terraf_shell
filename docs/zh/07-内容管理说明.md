# 07 — 内容管理说明

## 概述

本手册涵盖通过 OXP 管理后台管理的所有内容，包括材料库、文章、首页板块、法律页面和页脚内容。所有内容均支持英语、韩语和简体中文的多语言编辑。

---

## 1. 材料库

### 1.1 用途

材料库展示品牌旗舰可持续材料，是网站的科学和叙事核心。

**前台位置**：`/material`

### 1.2 材料字段

**位置**：管理后台 → CMS → 材料

| 字段 | 类型 | 多语言 | 说明 |
|---|---|---|---|
| `name` | 文本 | 是 | 材料名称 |
| `description` | 富文本 | 是 | 主要材料描述 |
| `story_cover_image` | 图片 | 否 | 材料故事版块封面图 |
| `internal initial-content marker` | 布尔值 | 否 | 标记初始内容 |

### 1.3 材料规格

**位置**：管理后台 → CMS → 材料规格

每个材料的技术参数表，每条规格包含：
- 所属材料
- 规格名称/标签（多语言）
- 规格值（多语言）
- 排序序号

### 1.4 故事章节

**位置**：管理后台 → CMS → 故事章节

讲述材料故事的叙事章节，每个章节包含：
- 所属材料
- 章节标题（多语言）
- 章节内容（多语言，富文本）
- 章节图片
- 排序序号

### 1.5 应用场景

**位置**：管理后台 → CMS → 应用场景

每个材料的使用案例展示，包含：
- 所属材料
- 应用标题（多语言）
- 应用描述（多语言）
- 应用图片
- 排序序号

## 2. 文章

### 2.1 用途

文章构成平台的知识库和博客，由管理员创作，涵盖材料、可持续发展和行业相关话题。

**前台位置**：`/articles`

### 2.2 文章字段

**位置**：管理后台 → CMS → 文章

| 字段 | 类型 | 多语言 | 说明 |
|---|---|---|---|
| `title` | 文本 | 是 | 文章标题 |
| `slug` | 文本 | 否 | URL 标识符（从标题自动生成） |
| `cover_image` | 图片 | 否 | 封面图片 |
| `content_json` | 富文本（JSON） | 否 | 完整文章内容（Tiptap 格式） |
| `excerpt` | 文本 | 是 | 文章摘要 |
| `reading_time` | 整数 | 否 | 预计阅读时间（分钟） |
| `status` | 枚举 | 否 | 已发布或草稿 |
| `published_at` | 时间戳 | 否 | 发布时间 |

### 2.3 创建文章

1. 前往**CMS → 文章 → 新建文章**。
2. 输入三种语言的**标题**。
3. 上传**封面图片**。
4. 使用富文本编辑器撰写文章内容。
5. 为每种语言添加**摘要**。
6. 准备好后将**状态**设为"已发布"。
7. 点击**保存**。

## 3. 首页板块

### 3.1 用途

首页由可配置的动态板块组成，无需代码更改即可更新首页内容。

**前台位置**：`/`（首页）

### 3.2 首页板块字段

**位置**：管理后台 → CMS → 首页板块

| 字段 | 类型 | 多语言 | 说明 |
|---|---|---|---|
| `section_type` | 文本 | 否 | 标识对应的前台渲染组件 |
| `title` | 文本 | 是 | 板块标题 |
| `subtitle` | 文本 | 是 | 板块副标题 |
| `content_json` | JSON | 否 | 板块内容（因板块类型而异） |
| `background_image` | 图片 | 否 | 背景图片 |
| `sort_order` | 整数 | 否 | 在首页的显示顺序 |
| `is_published` | 布尔值 | 否 | 是否在首页显示 |

### 3.3 管理首页内容

1. 前往 **CMS → 首页板块**。
2. 点击任意板块进行编辑。
3. 更新标题、副标题、图片或内容。
4. 调整 `sort_order` 重新排序各板块。
5. 切换 `is_published` 显示或隐藏板块。
6. 保存更改。

> **注意**：`section_type` 字段映射到具体的前台 React 组件。修改现有板块的 `section_type` 可能导致显示异常，建议在更改前咨询开发团队。

## 4. 法律页面

### 4.1 用途

隐私政策和使用条款页面可直接在管理后台编辑，通过 API 动态提供给前台。

**前台位置**：`/privacy`、`/terms`

### 4.2 编辑法律页面

**位置**：管理后台 → 系统 → 法律页面设置

- 分别编辑**隐私政策**和**使用条款**
- 支持三种语言（英/韩/中）的内容编辑
- 支持富文本编辑
- 每个页面有"最后更新"日期字段

### 4.3 注意事项

> **重要**：上线前必须由专业法律顾问审核法律文本。请勿在生产环境使用占位或草稿法律文本。

---

## 5. 多语言内容编辑

所有内容密集型字段支持三种语言。在管理后台编辑时，每种语言显示独立输入框（如 `name_en`、`name_ko`、`name_zh`）。

**备用语言规则**：若访客所选语言对应的翻译为空，系统自动回退至英文版本。

---

## 6. 图片和媒体管理

### 6.1 上传图片

在相关管理后台资源（产品、材料、文章、首页板块）中直接上传图片：
1. 点击图片上传字段。
2. 从本地选择文件。
3. 图片上传至 Azure Blob Storage（或本地磁盘）。
4. URL 或存储路径自动保存。

**支持格式**：JPG、JPEG、PNG、WebP、GIF

### 6.2 产品图片管理

**位置**：商城 → 产品图片

- 每个产品可上传多张图片
- 设置 `sort_order` 控制显示顺序
- 将一张图片标记为**主图**（作为默认显示图片）

### 6.3 媒体存储扫描

**位置**：管理后台 → 系统 → 媒体存储扫描

定期使用此工具：
- 识别存储中未被任何记录引用的孤立文件
- 发现引用了已删除文件的记录

## 7. SEO 字段

产品和文章包含 SEO 元数据字段：
- `seo_title` — 用于浏览器标签标题和 `<title>` 标签
- `seo_description` — 用于 `<meta name="description">` 标签

这些字段支持多语言，建议为每种语言填写，以优化不同市场的搜索引擎可见性。

---

## 8. 初始内容管理

初始内容通过 `php artisan db:seed` 初始化，所有初始项目标记有 `internal initial-content marker = true`。

**正式上线前**，通过**系统 → 初始内容清除**删除所有初始内容。

> **警告**：此操作不可撤销。删除前请确保真实内容已录入。

---

## 9. 内容管理最佳实践

1. **发布前填写所有语言版本**。
2. **使用有意义的别名（Slug）**（以英文为基础，跨语言共享）。
3. **上传前优化图片**（建议单张图片不超过 2 MB）。
4. **法律页面需专业审核**后方可发布。
5. **更改首页板块后测试显示效果**。
6. **归档而非删除**不再需要的文章和帖子——删除操作不可恢复。
7. **重大内容变更后备份设置**。

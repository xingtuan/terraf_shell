# 10 — 系统设置与配置说明

## 概述

OXP 有两层配置体系：

1. **环境级配置** — 通过 `.env` 文件设置，需重启服务生效，影响数据库、存储、邮件等底层配置。
2. **管理后台运行时配置** — 通过管理后台设置页面配置，存储于 `app_settings` 数据库表，修改后立即生效，无需重启。

---

## 1. 环境变量参考（`.env`）

文件位置：`B2C_backend/.env`

### 1.1 应用基础

| 变量 | 默认值 | 说明 |
|---|---|---|
| `APP_NAME` | `OXP` | 应用显示名称 |
| `APP_ENV` | `local` | 环境：`local`、`production` |
| `APP_KEY` | （生成） | Laravel 加密密钥——必须设置 |
| `APP_DEBUG` | `true` | 调试模式——**生产环境必须设为 `false`** |
| `APP_URL` | `http://127.0.0.1:8000` | 后端 URL——媒体服务依赖此值 |
| `APP_LOCALE` | `en` | 默认语言 |

### 1.2 数据库

| 变量 | 默认值 | 说明 |
|---|---|---|
| `DB_CONNECTION` | `mysql` | 数据库驱动 |
| `DB_HOST` | `127.0.0.1` | 数据库服务器地址 |
| `DB_PORT` | `3306` | 数据库端口 |
| `DB_DATABASE` | `product_community` | 数据库名称 |
| `DB_USERNAME` | `root` | 数据库用户名 |
| `DB_PASSWORD` | （空） | 数据库密码 |

### 1.3 文件存储

| 变量 | 默认值 | 说明 |
|---|---|---|
| `FILESYSTEM_DISK` | `azure` | 主存储磁盘：`azure` 或 `public` |
| `COMMUNITY_UPLOAD_DISK` | `azure` | 社区上传磁盘 |
| `AZURE_STORAGE_NAME` | （必填） | Azure Storage 账号名称 |
| `AZURE_STORAGE_KEY` | （必填） | Azure Storage 账号密钥 |
| `AZURE_STORAGE_CONTAINER` | `uploads` | Azure 容器名称 |
| `AZURE_STORAGE_URL` | （必填） | Azure Blob 端点 URL |
| `AZURE_STORAGE_USE_SAS_URLS` | `true` | 使用有时限 SAS URL |
| `AZURE_STORAGE_SAS_URL_TTL_MINUTES` | `10080` | SAS URL 有效期（7天） |
| `ALLOW_GUEST_UPLOAD` | `false` | 允许未认证用户上传文件 |

### 1.4 缓存与队列

| 变量 | 默认值 | 说明 |
|---|---|---|

| `SESSION_DRIVER` | `database` | 会话存储 |

### 1.5 邮件

| `MAIL_PORT` | `2525` | SMTP 端口（TLS 用 587，SSL 用 465） |

## 2. 管理后台运行时配置

### 2.1 邮件设置

**位置**：管理后台 → 系统 → 邮件设置

- SMTP 主机、端口、加密方式
- SMTP 用户名和密码
- 发件地址和发件人名称
- 测试邮件发送

> **注意**：运行时邮件设置优先于 `.env` 中的邮件配置。

### 2.2 运费设置

**位置**：管理后台 → 系统 → 运费设置

配置所有运费规则，包括标准运费、快递费、乡村附加费、免运费门槛和始发地。

### 2.3 税率设置

**位置**：管理后台 → 系统 → 税率设置

- GST 税率
- 税率标签（购物车和结账页显示）
- 价格是否含税

### 2.4 存储设置

**位置**：管理后台 → 系统 → 存储设置

在 Azure Blob Storage 和本地存储之间切换。

### 2.5 法律页面设置

**位置**：管理后台 → 系统 → 法律页面设置

编辑隐私政策和使用条款的内容（三种语言）。

### 2.6 社区设置

**位置**：管理后台 → 系统 → 社区设置

配置帖子提交策略。

### 2.7 功能开关

**位置**：管理后台 → 系统 → 功能开关

无需代码部署，直接开启或关闭特定平台功能。

## 3. 配置安全

- `app_settings` 表对敏感值使用**加密**存储。
- 不要直接在数据库中手动编辑 `app_settings` 表——值可能无法读取。
- `APP_KEY` 用于加密。该密钥变更后，所有加密设置将无法读取。
- 所有设置变更记录在 `app_setting_audit_logs` 审计日志中。

## 4. 设置备份与导入

**位置**：管理后台 → 系统 → 设置备份

**导出设置：**
1. 点击**导出设置**按钮。
2. 下载包含当前管理设置的 JSON 文件。
3. 安全存储备份文件。

**导入设置：**
1. 前往设置备份导入页面。
2. 上传之前导出的 JSON 文件。
3. 确认即将应用的变更。

> **注意**：导入设置将覆盖当前值，仅从可信来源导入。

---

## 5. 配置最佳实践

1. **绝不将 `.env` 提交到版本控制**——包含敏感信息。
2. **生产环境设置 `APP_DEBUG=false`**——调试模式会暴露敏感错误详情。
3. **生产环境设置 `APP_ENV=production`**。
4. **立即更改默认密码**——首次配置后立即更新管理员密码。
5. **正式发布前配置邮件**——`MAIL_MAILER=log` 开发模式不会发送真实邮件。
6. **设置正确的 CORS 域名**——生产环境 `CORS_ALLOWED_ORIGINS` 只设置前台实际域名。
7. **使用队列工作者**——生产环境运行 `php artisan queue:work` 确保后台任务可靠执行。
8. **定期备份设置**——每次重大变更后导出设置。

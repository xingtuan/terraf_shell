# 09 — 审核举报与风控管理

## 概述

OXP 平台内置完善的社区内容审核体系，以维护内容质量和用户安全。本手册涵盖从举报提交到处理完结的完整审核工作流程。

---

## 1. 举报管理

### 1.1 访问举报列表

**位置**：管理后台 → 社区 → 举报

举报列表显示所有内容举报：
- 举报人（用户名）
- 目标类型（帖子或评论）
- 被举报内容预览
- 举报状态
- 提交日期

**可用筛选：**
- 状态：待处理/审核中/已处理/已拒绝/已驳回
- 目标类型：帖子/评论
- 日期范围

### 1.2 处理举报步骤

1. **打开举报**查看详情。
2. **审查内容** — 点击链接查看被举报的帖子或评论。
3. **了解上下文** — 审查举报原因和相关对话。
4. **执行操作**：

**若内容不违规：**
- 点击**驳回** — 关闭举报，不采取操作。

**若内容违规：**
- 点击**审核中**更改状态。
- 选择处理操作：
  - **隐藏** — 从公开动态隐藏内容（可撤销）
  - **拒绝** — 标记内容为已拒绝（比隐藏更强）
  - **警告用户** — 向内容作者发送警告通知
  - **限制用户** — 禁止作者发帖或评论
  - **封禁用户** — 永久封禁作者账号

---

## 2. 审核操作参考

### 2.1 隐藏内容

- 帖子或评论从公开动态中消失
- 作者仍可看到自己的内容
- 可撤销，是最轻度的操作
- 适用于：需要临时隐藏但可能误判的内容

### 2.2 拒绝内容

- 内容标记为已拒绝
- 从公开动态移除
- 记录于审核日志
- 适用于：明确违规需要清除的内容

### 2.3 警告用户

- 向内容作者发送系统通知
- 内容本身不受影响
- 创建违规记录
- 适用于：首次或轻微违规

### 2.4 限制用户

- 账号状态变更为 `restricted`（受限）
- 用户仍可登录和浏览，但无法发帖或评论
- 可撤销（将账号状态改回正常）
- 适用于：多次轻微违规的用户

### 2.5 封禁用户

- 账号状态变更为 `banned`（封禁）
- 用户无法登录
- 永久性操作，应谨慎使用
- 适用于：严重违规行为

## 3. 用户违规记录

**位置**：管理后台 → 社区 → 用户违规

用户违规记录存储违规的正式记录：

| 字段 | 说明 |
|---|---|
| 用户 | 违规用户 |
| 违规类型 | 骚扰/仇恨言论/垃圾内容/虚假信息/版权侵权/其他 |
| 严重程度 | 低/中/高/严重 |
| 状态 | 有效/已处理/已申诉 |
| 说明 | 违规详情 |

### 3.1 严重程度建议操作

| 严重程度 | 典型操作 |
|---|---|
| 低 | 警告 |
| 中 | 临时限制 |
| 高 | 延长限制或封禁 |
| 严重 | 立即封禁 |

---

## 4. 审核日志

**位置**：管理后台 → 社区 → 审核日志

每次审核操作均记录在 `moderation_logs` 表：
- 执行操作的管理员/版主
- 操作对象（帖子、评论或用户）
- 采取的操作
- 日期时间
- 附加备注

---

## 5. 管理员操作日志

**位置**：管理后台 → 社区 → 管理员操作日志

涵盖所有管理员操作的更广泛审计日志，包括：
- 设置变更
- 用户角色修改
- 内容状态变更
- 系统操作

---

## 6. 内容直接审核（无需举报）

管理员也可直接审核内容，无需等待用户举报：

**帖子直接审核**：
- 在**社区 → 帖子**中直接更改帖子状态为"存档"或"已拒绝"
- 切换"精选"和"置顶"状态

**评论直接审核**：
- 在**社区 → 评论**中更改评论状态或删除评论

---

## 7. 社区审核设置

**位置**：管理后台 → 系统 → 社区审核设置

| 设置 | 选项 | 效果 |
|---|---|---|
| 提交策略 | 自动发布/需要审核/限制发布 | 控制新帖子的处理方式 |
| 敏感词过滤 | 启用/禁用 | 启用可配置的词语屏蔽列表 |
| 敏感词列表 | 逗号分隔的词语 | 触发内容标记的词语 |

## 8. 版主角色说明

`moderator`（版主）角色有权进行审核操作，但不具备完整的管理员权限：

**版主可以：**
- 审核和处理举报
- 更改帖子和评论状态
- 警告、限制或封禁用户

**版主不能：**
- 访问产品/商城管理
- 访问财务或系统设置
- 创建或删除分类

通过**管理后台 → 用户 → 用户 → 编辑用户 → 角色 → 版主**分配版主角色。

---

## 9. 审核最佳实践

1. **保持一致**：对所有举报适用相同标准，不因举报人或内容作者身份而有所不同。
2. **记录决策**：驳回边界案例时添加备注，便于日后参考。
3. **及时上报**：威胁内容、违法内容或严重骚扰应立即上报管理员。
4. **复核封禁**：重要账号的封禁决策建议由至少两名管理员审批。
5. **关注规律**：同一用户积累多次警告应提升至限制处理。
6. **通知举报人**：在政策允许的范围内，向举报人反馈处理结果。
7. **保持日志清晰**：每次操作都应附有说明。

# 08 — 订单与客户管理

## 概述

本手册从管理员视角介绍订单处理、客户账号管理和地址管理。

---

## 1. 订单管理

### 1.1 订单列表

**位置**：管理后台 → 商城 → 订单

订单列表显示所有订单，包含：
- 订单编号（格式：`OXP-XXXXXX`）
- 客户（注册用户显示姓名和邮箱；游客显示游客邮箱）
- 订单总额
- 订单状态
- 支付状态
- 下单日期

**筛选选项：**
- 按订单状态筛选（待处理、已确认、处理中、已发货、已送达、已取消）
- 按支付状态筛选（未支付、已支付、已退款、支付失败）
- 按订单编号或客户邮箱搜索

### 1.2 订单详情页

点击任意订单查看完整详情：

**订单信息**：订单编号、下单时间、客户类型（游客/注册用户）、客户邮箱

**订单商品**：商品变体名称和 SKU、数量、下单时的单价快照、行总价

**财务汇总**：小计、运费、税额（GST）、订单总额

**配送信息**：完整收货地址快照、所选运费方式、运费报价快照

**状态信息**：当前订单状态、各状态时间戳、当前支付状态

### 1.3 更新订单状态

从订单详情页点击**编辑**：
- 更新**订单状态**
- 更新**支付状态**（收款确认后）
- 记录**支付参考号**

**建议处理流程：**
1. 新订单默认**待处理** — 尽快核实并确认。
2. 设为**已确认**表示认可收单。
3. 开始备货时设为**处理中**。
4. 包裹寄出后设为**已发货**。
5. 确认收货后设为**已送达**。

---

## 2. 订单状态参考

| 状态 | 使用时机 | 客户看到的内容 |
|---|---|---|
| **待处理** | 已下单，尚未审核 | 订单已收到 |
| **已确认** | 管理员确认 | 订单已确认 |
| **处理中** | 拣货打包中 | 备货中 |
| **已发货** | 已发出包裹 | 已发货 |
| **已送达** | 确认签收 | 已送达 |
| **已取消** | 订单取消 | 已取消 |

---

## 3. 支付状态参考

| 状态 | 说明 |
|---|---|
| **未支付** | 默认状态，未收到付款 |
| **已支付** | 管理员确认收款 |
| **已退款** | 已办理退款 |
| **支付失败** | 支付尝试失败 |

> **注意**：平台不含自动支付网关，所有支付状态变更需人工操作。

---

## 4. 游客订单追踪

游客无需账号即可查询订单：
- **前台路径**：`/store/orders`
- **所需信息**：订单编号（OXP-XXXXXX）+ 结账时使用的邮箱地址

管理员可在订单列表中查看所有游客订单（通过无关联账号识别，仅有游客邮箱）。

---

## 5. 客户账号管理

### 5.1 查看客户账号

**位置**：管理后台 → 用户 → 用户

按角色筛选查看普通客户（角色：`creator`）。每个客户资料显示：
- 注册日期
- 账号状态
- 个人资料信息
- 关联订单
- 社区活动记录

### 5.2 账号状态管理

若客户反映账号问题，管理员可：
- 将账号状态重置为**正常**（如误封）
- **暂停**活动异常的账号（临时）
- **封禁**违规账号（永久）
- **限制**账号（保留购物权限，禁止发帖）

## 6. 常见订单处理场景

### 6.1 已下单但未支付

1. 与客户确认支付状态。
2. 在支付平台核查交易记录。
3. 确认付款：将 `payment_status` 更新为"已支付"。
4. 支付失败：更新为"支付失败"，请客户重新支付。

### 6.2 客户申请取消

1. 若订单处于**待处理**或**已确认**：可直接取消。
2. 若订单处于**处理中**：评估是否仍可拦截。
3. 若订单已**发货**：无法召回，待签收后处理退货/退款。

### 6.3 地址填写错误

1. 若订单仍为**待处理**或**已确认**：联系客户确认正确地址后可更新。
2. 注意：订单上的收货地址为快照，不会随客户更新收货地址自动变更。

### 6.4 下单后库存不足

1. 联系客户告知延迟情况或提供替代选项。
2. 更新订单状态反映实际情况。
3. 若客户选择退款，人工通过支付平台处理。

---

## 7. 库存盘点建议

处理订单后手动更新库存：

1. 前往**管理后台 → 商城 → 库存**。
2. 找到对应商品变体。
3. 点击**编辑**。
4. 更新 `stock_quantity` 字段。
5. 记录库存调整备注。

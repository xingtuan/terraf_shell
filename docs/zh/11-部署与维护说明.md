# 11 — 部署与维护说明

## 概述

本指南涵盖 OXP 平台的本地开发环境搭建、生产环境部署和日常维护操作。

---

## 1. 系统要求

### 1.1 后端要求

| 组件 | 最低版本 | 推荐版本 |
|---|---|---|
| PHP | 8.2 | 8.3+ |
| MySQL | 8.0 | 8.0+ |
| Composer | 2.x | 最新 2.x |
| Node.js | 18.x | 20 LTS |

必要 PHP 扩展：`pdo_mysql`、`mbstring`、`openssl`、`json`、`fileinfo`、`gd`（或 `imagick`）、`bcmath`

---

## 2. 本地开发环境搭建

### 2.1 后端搭建

```bash
# 1. 克隆仓库
git clone {仓库地址}
cd terraf/B2C_backend

# 2. 安装 PHP 依赖
composer install

# 3. 复制环境配置文件
cp .env.example .env

# 4. 生成应用密钥
php artisan key:generate

# 5. 配置 .env（填写数据库等配置）

# 6. 运行数据库迁移
php artisan migrate

# 7. 填充初始内容（可选）
php artisan db:seed

# 8. 创建存储符号链接（使用本地存储时）
php artisan storage:link

# 9. 启动开发服务器
php artisan serve
```

后端运行于 `http://127.0.0.1:8000`。

### 2.2 前端搭建

```bash
cd terraf/B2C_frontend

npm install

# 创建 .env.local 文件，配置：
# NEXT_PUBLIC_API_URL=http://127.0.0.1:8000/api

npm run dev
```

前端运行于 `http://localhost:3000`。

### 2.3 开发环境说明

- 邮件设置 `MAIL_MAILER=log` — 邮件内容写入 `storage/logs/laravel.log`
- 存储设置 `FILESYSTEM_DISK=public` — 使用本地存储代替 Azure
- 队列设置 `QUEUE_CONNECTION=sync` — 同步执行任务（无需队列工作者）

## 3. 生产环境部署

### 3.1 后端部署

```bash
# 设置生产环境
APP_ENV=production
APP_DEBUG=false

# 安装依赖（不含开发包）
composer install --no-dev --optimize-autoloader

# 配置 .env（生产环境配置）

# 执行数据库迁移
php artisan migrate --force

# 优化 Laravel
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan optimize

# 设置文件权限
chmod -R 775 storage
chmod -R 775 bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
```

### 3.2 前端生产构建

```bash
cd B2C_frontend

npm install

# 创建 .env.production，配置：
# NEXT_PUBLIC_API_URL=https://api.你的域名.com

npm run build
npm run start
```

## 4. 队列工作者配置

平台使用 Laravel 队列处理后台任务（主要是通知）。生产环境需持续运行队列工作者。

### 4.1 Supervisor 配置（推荐）

```ini
[program:oxp-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/B2C_backend/artisan queue:work --sleep=3 --tries=3
autostart=true
autorestart=true
user=www-data
numprocs=2
stdout_logfile=/var/www/B2C_backend/storage/logs/worker.log
```

```bash
supervisorctl reread
supervisorctl update
supervisorctl start oxp-worker:*
```

## 5. 日常维护命令

### 5.1 清除缓存

```bash
# 清除全部缓存
php artisan optimize:clear

# 修改 .env 后必须运行此命令
```

### 5.2 数据库迁移

```bash
# 查看迁移状态
php artisan migrate:status

# 执行待处理的迁移
php artisan migrate
```

### 5.3 队列管理

```bash
# 查看失败任务
php artisan queue:failed

# 重试全部失败任务
php artisan queue:retry all

# 清除失败任务
php artisan queue:flush
```

### 5.4 日志清理

```bash
php artisan log:clear
```

## 6. 更新项目

### 6.1 后端更新

```bash
git pull origin main
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan optimize:clear
php artisan optimize
supervisorctl restart oxp-worker:*
```

### 6.2 前端更新

```bash

git pull origin main
npm install

# 重启前端服务
```

## 7. 数据库备份策略

```bash
# 手动备份
mysqldump -u 用户名 -p 数据库名 > backup_$(date +%Y%m%d).sql

# 定时任务（加入 crontab）
0 2 * * * mysqldump -u 用户名 -p密码 数据库名 | gzip > /backups/db_$(date +\%Y\%m\%d).sql.gz
```

**建议备份保留期限：**
- 每日备份：保留 30 天
- 每周备份：保留 12 周
- 每月备份：保留 12 个月
- 备份存储于服务器之外的独立位置

---

## 8. 媒体备份策略

**使用 Azure Blob Storage：**
- Azure 提供内置冗余
- 定期使用 AzCopy 或 Azure 备份功能快照容器

**使用本地存储：**
- 将 `storage/app/public/` 纳入日常备份计划

---

## 9. 维护模式

```bash
# 启用维护模式
php artisan down --message="系统维护中，请稍后再访问。"

# 执行部署操作...

# 关闭维护模式
php artisan up
```

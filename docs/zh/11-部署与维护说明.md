## 推荐部署方式

当前项目推荐使用根目录 `auto_deploy.sh` 在 Ubuntu / Debian 单机服务器上部署：

```bash
sudo bash auto_deploy.sh your-domain-or-ip
```

常用生产更新命令：

```bash
sudo env RUN_SEED=0 bash auto_deploy.sh your-domain-or-ip
```

脚本会安装 PHP 8.3、Composer、Node.js 20、pnpm、MySQL、Nginx、PHP-FPM、Supervisor 和 Cron，并完成代码拉取、依赖安装、数据库迁移、前端构建、队列和调度器配置。

## 访问地址

默认部署完成后：

- 前端：`http://your-domain-or-ip/`
- API：`http://your-domain-or-ip/api/`
- 后台：`http://your-domain-or-ip:8000/admin`
- 健康检查：`http://your-domain-or-ip:8000/up`

部署凭据和常用命令会写入：

```text
/root/terraf-install/credentials.txt
```

## 重新部署

普通重新部署：

```bash
sudo env RUN_SEED=0 bash auto_deploy.sh your-domain-or-ip
```

如果脚本提示已有仓库存在本地修改，先查看：

```bash
git -C /var/www/terraf_shell status --short
```

确认可以丢弃本地修改后才使用：

```bash
sudo env RUN_SEED=0 RESET_WORKTREE=1 bash auto_deploy.sh your-domain-or-ip
```

`RESET_WORKTREE=1` 会执行 Git 硬重置并删除未跟踪文件，不能随意使用。

## 维护重点

- 上线后修改默认管理员密码。
- 生产环境使用 `RUN_SEED=0` 更新，除非明确需要重新执行 Seeder。
- 定期备份 MySQL 数据库和上传文件。
- 检查 `terraf-frontend`、`terraf-queue`、Nginx、PHP-FPM、MySQL。
- 配置 HTTPS 后同步更新 `APP_URL`、`FRONTEND_URL`、`NEXT_PUBLIC_SITE_URL`、CORS 和 Sanctum 域名。
- 使用后台 Storage Settings 测试 local / Azure storage。
- 使用后台 Tax Settings 和 Shipping Settings 校验 GST 与配送费。

## 常用命令

```bash
sudo systemctl status terraf-frontend
sudo supervisorctl status terraf-queue:*
sudo systemctl status nginx
sudo systemctl status php8.3-fpm
sudo systemctl status mysql
```

清缓存：

```bash
cd /var/www/terraf_shell/B2C_backend
php artisan optimize:clear
php artisan config:cache
```

重建前端：

```bash
cd /var/www/terraf_shell/B2C_frontend
pnpm build
sudo systemctl restart terraf-frontend
```

# Internationalization

The system supports English, Chinese, and Korean. Frontend messages, backend API messages, validation messages, seed data, and admin labels should remain aligned across all three locales.

## Locales

Current locale codes:

- `en`
- `zh`
- `ko`

Frontend routes are locale-scoped, for example:

```text
/en/store
/zh/store
/ko/store
```

## Frontend Messages

Frontend translation files:

```text
B2C_frontend/messages/en.json
B2C_frontend/messages/zh.json
B2C_frontend/messages/ko.json
```

Locale configuration:

```text
B2C_frontend/lib/i18n.ts
```

When adding frontend text, update all three JSON files and run:

```bash
cd B2C_frontend
node scripts/check-i18n-keys.mjs
node scripts/i18n-diff.mjs
```

`pnpm build` runs `scripts/i18n-diff.mjs` before `next build`.

## Backend Language Files

Backend translations:

```text
B2C_backend/lang/en
B2C_backend/lang/zh
B2C_backend/lang/ko
```

Current groups:

- `admin.php`
- `api.php`
- `auth.php`
- `validation.php`

Add API, admin, and validation strings to all three locales.

## Validation Messages

Laravel validation messages come from `lang/{locale}/validation.php`. Avoid hard-coded English errors when a translation key is available.

When adding fields, confirm:

- Field names are readable.
- Custom errors are translated.
- The frontend can display the returned errors.

## Admin Panel Translations

Filament labels use backend `lang/{locale}/admin.php` plus Filament vendor translations. Locale switching is exposed in the admin user menu.

Check admin translations:

```bash
cd B2C_backend
php artisan admin:check-translations
```

When adding resources, pages, actions, or fields, translate navigation labels, model labels, form labels, table columns, action labels, helper text, notifications, and validation text.

## Seed Data

Seeders should provide English, Chinese, and Korean values for:

- Product categories.
- Product names and descriptions.
- Product attributes and values.
- Material content.
- Page sections.
- Articles / CMS content.
- Email templates.

`ProductCatalogSeeder` creates official starter catalog content.

## Adding A Translatable Field

Recommended checklist:

1. Add the database migration.
2. Update model fillable / casts.
3. Update API resource output.
4. Update admin form and table.
5. Update frontend types and UI.
6. Add frontend messages in `en.json`, `zh.json`, and `ko.json`.
7. Add backend/admin language keys in `lang/en`, `lang/zh`, and `lang/ko`.
8. Update seed data.
9. Run i18n checks and relevant tests.

## Fallbacks

Some frontend and backend displays may fall back when a locale is missing. Production-facing content should not rely on fallback behavior; complete all three locales before handover.

## Commands

```bash
cd B2C_frontend
node scripts/check-i18n-keys.mjs
node scripts/i18n-diff.mjs
pnpm build

cd ../B2C_backend
php artisan admin:check-translations
```

## Common Issues

- Missing frontend key: run `node scripts/check-i18n-keys.mjs`.
- Admin still shows English: check translation keys and hard-coded labels.
- Validation text uses the wrong language: check locale middleware and `validation.php`.
- Seed data is incomplete: update seeders and evaluate whether production reseeding is safe.

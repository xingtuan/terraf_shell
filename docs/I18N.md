# 国际化说明

系统支持英文、中文、韩文三语。前端、后端 API 文案、验证消息、Seed 数据和管理后台主要文案都应保持三语一致。

## 语言代码

当前语言代码：

- `en`
- `zh`
- `ko`

前端路由按 locale 组织，例如：

```text
/en/store
/zh/store
/ko/store
```

## 前端翻译文件

前端翻译文件位于：

```text
B2C_frontend/messages/en.json
B2C_frontend/messages/zh.json
B2C_frontend/messages/ko.json
```

语言配置位于：

```text
B2C_frontend/lib/i18n.ts
```

新增前端文案时必须同步三个 JSON 文件，并运行检查：

```bash
cd B2C_frontend
node scripts/check-i18n-keys.mjs
node scripts/i18n-diff.mjs
```

`pnpm build` 会先运行 `scripts/i18n-diff.mjs`。

## 后端翻译文件

后端翻译文件位于：

```text
B2C_backend/lang/en
B2C_backend/lang/zh
B2C_backend/lang/ko
```

当前包含：

- `admin.php`
- `api.php`
- `auth.php`
- `validation.php`

新增 API 错误、后台标签或验证消息时，应同步三语文件。

## Validation Message

Laravel 验证消息来自 `lang/{locale}/validation.php`。接口返回错误时，应避免硬编码英文句子，优先使用 Laravel 翻译 key。

新增字段时需要确认：

- 属性名是否在 validation attributes 中可读。
- 自定义错误是否同步三语。
- 前端是否能显示错误信息。

## Admin Panel 翻译

Filament 后台主要文案位于后端 `lang/{locale}/admin.php` 以及 Filament vendor 翻译。后台语言切换通过 `/admin/locale/{locale}` 和用户菜单操作实现。

检查后台翻译：

```bash
cd B2C_backend
php artisan admin:check-translations
```

新增 Resource、Page、Action 或 Form 字段时，应同步：

- 导航标签。
- Model label。
- Form label。
- Table column。
- Action label。
- 帮助文本。
- 提示和通知。

## Seed 数据多语言

Seeders 中的内容应提供三语字段，尤其是：

- 商品分类。
- 商品标题和描述。
- 商品属性和值。
- 材料内容。
- 页面 Section。
- 文章 / CMS 内容。
- 邮件模板。

当前 `ProductCatalogSeeder` 创建的是正式初始化商品目录，文档应按正式初始化内容描述。

## 新增字段适配三语

新增需要展示给用户或管理员的字段时，按以下步骤处理：

1. 数据库迁移新增字段。
2. Model fillable / cast 更新。
3. API Resource 或 Transformer 输出字段。
4. 后台 Form / Table 增加字段。
5. 前端类型和 UI 增加字段。
6. `messages/en.json`、`messages/zh.json`、`messages/ko.json` 增加前端文案。
7. `lang/en`、`lang/zh`、`lang/ko` 增加后端和后台文案。
8. Seeder 同步三语初始化数据。
9. 运行 i18n 检查和相关测试。

## 回退策略

前端和后端在部分内容缺少某语言时可能展示 fallback 内容，但交付文档、正式初始化数据和客户可见内容不应依赖 fallback。上线前应检查三语完整性。

## 常用命令

前端 key 检查：

```bash
cd B2C_frontend
node scripts/check-i18n-keys.mjs
node scripts/i18n-diff.mjs
```

前端构建：

```bash
cd B2C_frontend
pnpm build
```

后台翻译检查：

```bash
cd B2C_backend
php artisan admin:check-translations
```

## 常见问题

### 前端某语言缺 key

运行 `node scripts/check-i18n-keys.mjs`，根据输出补齐三个 JSON 文件。

### 后台仍显示英文

检查对应 Resource / Page 是否使用 translation key，或是否缺少 `lang/{locale}/admin.php` 条目。

### 验证消息不跟随语言

检查请求 locale、Laravel middleware、`validation.php` 和前端错误展示逻辑。

### Seed 数据只有英文

补齐 Seeder 的三语字段，重新执行前确认是否会覆盖已有正式内容。生产环境不要盲目 `db:seed --force`。
